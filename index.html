引出动态规划问题

在这里，我们使用一个硬币找零问题引出动态规划问题。

问题描述：
给定一组硬币数，找出一组有多少种组合，来找换零钱N。

比如，可用来找零的硬币为: 1、3、4 待找的钱数为 6。用两个面值为3的硬币找零，最少硬币数为2。而不是 4，1，1

思路解析：
本题十分经典，可以使用逐步递进的思路，来一步一步地改进自己的解法：从
暴力搜索法—>记忆搜索法—>动态规划法—>状态继续简化之后的动态规划法
，并且可以从其中了解到思考出动态规划方法的过程。

1. 暴力搜索法

arr = {5、10、25、1} ， aim = 1000

  1. 用0张5元货币，让{10、25、1}组成剩下的1000元，最终方法记为—res1
  2. 用1张5元货币，让{10、25、1}组成剩下的995元，最终方法记为—res2
  3. 用2张5元货币，让{10、25、1}组成剩下的990元，最终方法记为—res3
    ….
    201.用200张5元货币，让{10、25、1}组成剩下的0元，最终方法记为—res201

所以，最终结果res = res1 + res2 + res3 + … + res201 (累加即是结果)

定义递归函数：int p1(arr, index, aim)它的含义是用arr[index … N-1]这些面值的钱组成剩下的aim时，所有的方法数，采用递归地方式进行计算，可能对时间复杂度不太友好。

代码：

// 暴力搜索
public int coins1(int[] arr, int aim) {
    if (arr == null || arr.length == 0 || aim < 0) {
        return 0;
    }
    return process1(arr, 0, aim);
}

private int process1(int[] arr, int index, int aim) {
    int res = 0;
    if (index == arr.length) {
        res =  aim == 0 ? 1 : 0;
    } else {
        for (int i = 0; i * arr[index] <= aim; i++) {
            res += process1(arr, index + 1, aim - i * arr[index]);
        }
    }
    return res;
}

2. 记忆搜索方法

上面的暴力搜索会有大量重复的计算过程。比如说：当选择0张5元，1张10元，那么接下来的剩余解法过程，与2张5元，0张10元，的剩余的解法数量，是一样，但是暴力搜索不会理解，它会重复进行计算。这样冗余的计算大量存在，使得暴力搜索时间复杂度极高。

为了优化这一过程，我们可以采用记忆搜索法，使用一个哈希表（数组或者Map）组将计算过的数据进行保存。
将递归股构成中变化的部门总结出来，然后进行递归，p(index, aim)。

  1. 每计算完一个p(index, aim)，都将结果放入map中，index和aim组成共同的key，返回结果作为value。
  2. 要进入一个递归过程p(index, aim)，先以index和aim注册的key在map中查询是否已经存在value，如果存在value，则直接取值，如果不存在，才进行递归。

代码：

// 记忆搜索
public int coins2(int[] arr, int aim) {
    if (arr == null || arr.length == 0 || aim < 0) {
        return 0;
    }

    int[][] map = new int[arr.length + 1][aim + 1];
    return process2(arr, 0, aim, map);
}

private int process2(int[] arr, int index, int aim, int[][] map) {
    int res = 0;
    if (index == arr.length) {
        res =  aim == 0 ? 1 : 0;
    } else {
        int mapValue = 0;
        for (int i = 0; i * arr[index] <= aim; i++) {
            mapValue = map[index + 1][aim - i * arr[index]];
            if (mapValue != 0) {
                res += mapValue == - 1 ? 0 : mapValue;
            } else {
                res += process2(arr, index + 1, aim - i * arr[index], map);
            }
        }
        // 每次计算完，将计算结果存入上一位相应位置，进行保存结果， -1代表没有这种解法
        map[index][aim] = res == 0 ? - 1 : res;
    }
    return res;
}

3. 经典动态规划

如果arr长度为N，生成行数为N，列数为aim + 1的矩阵dp[i][j]的含义就是在使用arr[0…i]货币的情况下，组成钱数有多少种方法。

（Pic Here）

其实：

  1. 记忆化搜索法就是某种形态的动态规划法
  2. 记忆化搜索法不关心到达某个递归过程的路径，而只是单纯地对计算过的递归规程进行记录，避免重复地递归过程。
  3. 动态规划的方法是规定好每一个递归过程的计算顺序，依次进行计算，后面的计算严格依赖前面的计算过程。
  4. 两者都是空间换时间的方法，也都是枚举的过程，区别就是动态规划规定了计算顺序。

dp[i][j] = dp[i - 1][j] + dp[i - 1][j - 1 arr[i]] + dp[i - 1][j - 2 arr[i]] + … + dp[i - 1][0]

什么是动态规划？

  1. 其本质就是申请空间记录每一个暴力搜索的结果，下次要用结果的时候直接使用，而不再重复地递归
  2. 动态规划规定每一种递归状态的计算顺序，依次进行顺序
  3. 动态规划相比记忆搜索法有计算顺序，这样就有了进一步优化的可能性

这里的这里可有优化为：

dp[i][j] = dp[i - 1][j] + dp[i - 1][j - 1 arr[i]] + dp[i - 1][j - 2 arr[i]] + … + dp[i - 1][0] = dp[i - 1][j] + （dp[i - 1][j - 1 arr[i]] + dp[i - 1][j - 2 arr[i]] + … + dp[i - 1][0]） = dp[i - 1][j] + dp[i][j - arr[i]]

以后遇到暴力递归问题思路：

  1. 实现暴力递归方法
  2. 在暴力递归方法中找到递归过程
  3. 找到递归过程的参数之后，记忆化搜索方法很好找到
  4. 通过分析记忆化搜索的依赖路径，进而实现动态规划
  5. 通过记忆化搜索改出动态规划，进而看看能否进行化简，如果能化简，还能实现时间复杂度更低的动态规划方法。

这里优化过后的实现代码如下：

        }
    }

    return dp[n - 1][aim];
}

动态规划关键点：

  1. 最优化原理。
  2. 无后效性。
  3. 子问题的重叠性（解决冗余）

下面是一些经典问题：

1.矩阵最小路径问题

题目：有一个矩阵map，它每个格子有一个权值。从左上角的格子开始每次只能向右或者向下走，最后到达右下角的位置，路径上所有的数字累加起来就是路径和，返回所有的路径中最小的路径和。

给定一个矩阵map及它的行数n和列数m，请返回最小路径和。保证行列数均小于等于100.

测试样例： [[1,2,3],[1,1,1]],2,3
返回：4

思路：dp[i][j]代表到达第i行第j列时的的路径总和，因为每次只能往下走或者往右走，所以非第一行或者第一列递推公式为： dp[i][j] = max{dp[i - 1][j] , dp[i][j - 1]} + map[i][j];

代码如下：

public class MinimumPath {
public int getMin(int[][] map, int n, int m) {
    // write code here
    // 使用dp，dp[i][j]代表到达第i行第j列时的的路径总和
    int[][] dp = new int[n][m];

    dp[0][0] = map[0][0];
    // 先初始化
    for (int i = 1; i < n; i++) {
        dp[i][0] = dp[i - 1][0] + map[i][0];
    }
    for (int i = 1; i < m; i++) {
        dp[0][i] = dp[0][i - 1] + map[0][i];
    }

    for (int i = 1; i < n; i++) {
        for (int j = 1; j < m; j++) {
            dp[i][j] = dp[i - 1][j] < dp[i][j - 1] ? 
                    dp[i -1][j] + map[i][j] : dp[i][j - 1] + map[i][j];
        }
    }

    return dp[n - 1][m - 1];
}
}

2.经典问题（最长递增子序列—LIS）

问题:这是一个经典的LIS(即最长上升子序列)问题，请设计一个尽量优的解法求出序列的最长上升子序列的长度。

给定一个序列A及它的长度n(长度小于等于500)，请返回LIS的长度。

测试样例： [1,4,2,5,3],5
返回：3

思路：使用一个状态数组dp[n]来盛放遍历到位置i时的LIS的值，对i的每一次循环维护一个
max = 0值，状态转移方程式：dp[i] = A[i] > A[j] ? max{dp[j], max} + 1 : max + 1;
然后从dp[i]中找出最大的值即为结果

代码：

public class LongestIncreasingSubsequence {
    public static int getLIS(int[] A, int n) {
        // write code here
        // 代表在i处的最长上升子序列
        int[] dp = new int[n];
        dp[0] = 1;

        int res = Integer.MIN_VALUE;
        for (int i = 1; i < n; i++) {
            int max = 0;
            for (int j = 0; j < i; j++) {
                if (A[j] < A[i]) {
                    max = max > dp[j] ? max : dp[j];
                }
            }
            dp[i] = max + 1;
            res = res > dp[i] ? res : dp[i];
        }

        return res;
    }
}

3.经典问题（最长公共子序列—LIS）

问题：给定两个字符串A和B，返回两个字符串的最长公共子序列的长度。

例如，A=”1A2C3D4B56”，B=”B1D23CA45B6A”，”123456”或者”12C4B6”都是最长公共子序列。

给定两个字符串A和B，同时给定两个串的长度n和m，请返回最长公共子序列的长度。

保证两串长度均小于等于300。

测试样例： “1A2C3D4B56”,10,”B1D23CA45B6A”,12

返回：6

思路：构造一个二维数组dp[n][m]，dp[i][j]代表字符串A的0…i-1，匹配字符串B的0…j-1时的最长公共子
序列，对于dp[i][j]，最长公共子序列只有三种情况：

1. 字符串A的0...i-2与B的0...j-1的最长公共子串（字符串A的位置i-1与B的位置j-1字符不相等）
2. 字符串A的0...i-1与B的0...j-2的最长公共子串（字符串A的位置i-1与B的位置j-1字符不相等）
3. 字符串A的位置i-1与B的位置j-1字符相等，等于A的0...i-2与B的0...j-2的最长公共子串加一

注释：这里构造第一行和第一列的时候需要注意，当第一次出现字符相等得时候，之后的最长公共子序列就不用判断，直接赋值1。

代码：

public class LCS {
    public static void main(String[] args) {
        String a = "1A2C3D4B56";
        String b = "B1D23CA45B6A";
        System.out.println(findLCS(a, a.length(), b, b.length()));
    }

    public static int findLCS(String A, int n, String B, int m) {
        int[][] dp = new int[n][m];

        boolean flag = false;
        for (int i = 0; i < n; i++) {
            if (!flag) {
                if (A.charAt(i) == B.charAt(0)) {
                    dp[i][0] = 1;
                    flag = true;
                }
            } else {
                dp[i][0] = 1;
            }
        }

        flag = false;
        for (int i = 0; i < m; i++) {
            if (!flag) {
                if (B.charAt(i) == A.charAt(0)) {
                    dp[0][i] = 1;
                    flag = true;
                }
            } else {
                dp[0][i] = 1;
            }
        }


        for (int i = 1; i < n; i++) {
            for (int j = 1; j < m; j++) {
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
                if (A.charAt(i) == B.charAt(j)) {
                    dp[i][j] = Math.max(dp[i - 1][j - 1] + 1, dp[i][j]);
                }
            }
        }
        return dp[n - 1][m - 1];
    }
}

4.经典问题（0-1背包）

问题：一个背包有一定的承重cap，有N件物品，每件都有自己的价值，记录在数组v中，也都有自己的重量，记录在数组w中，每件物品只能选择要装入背包还是不装入背包，要求在不超过背包承重的前提下，选出物品的总价值最大。

给定物品的重量w价值v及物品数n和承重cap。请返回最大总价值。

测试样例： [1,2,3],[1,2,3],3,6
返回：6

思路：根据递归公式：创建一个二维数组dp[n][cap+1]，dp[i][j]代表当0…i-1 + 1件物品被选择
放在一个承重为j的背包中，最大价值0…cap，那么对于dp[i][j]，有多种情况：
1.对于一个背包承重的重量不足以存放第i个物品，即为j < w[i]，那么就用剩余的0 .. i-1来完成背包
2.对于一个背包承重的重量足以存放第i个物品，即为j > w[i]，那么就从两种情况里面取最大值：

    - 剩余的0 .. i-1来完成背包的重量，此时的总价值
    - 放下w[i]之后，在把剩余的j-w[i]用来0 .. i-1来完成背包剩余的重量，此时的总价值。
状态转移方程： dp[i][j] = j > w[i] ? max(dp[i-1][j-w[i]] + v[i], dp[i-1][j]) : dp[i-1][j];

代码：

public class Backpack {
    public static int maxValue(int[] w, int[] v, int n, int cap) {
    int[][] dp = new int[n][cap + 1];
    // 初始化第一列，dp[i][0]，代表选择总质量为0的时候，选择的物品总价值
    // 初始化第一行，dp[0][i]，代表选择第1件物品时，总重量为y的时候，选择物品最大价值
    for (int i = 0; i < cap + 1; i++) {
        if (i >= w[0]) {
            dp[0][i] = v[0];
        }
    }

    // dp数组初始化，当i==0或者j==0时，结果为0
    for (int i = 1; i < n; i++) {
        for (int j = 1; j <= cap; j++) {
            if (w[i] > j) {
                dp[i][j] = dp[i - 1][j];
            } else {
                dp[i][j] = Math.max(dp[i - 1][j - w[i]] + v[i], dp[i - 1][j]);
            }
        }

    }
    return dp[n - 1][cap];
}

优化：动态规划的思路就是用空间换时间。但是有时候经典动态规划的空间复杂度可以优化。
相比记忆搜索法，这是动态规划严格按照步骤来递推的好处。
在此我们观察到：

  1. 每一轮的第i个行的值一定来自第i-1行第j列当前位置dp[i - 1][j]
  和前面的值(dp[i - 1][j - w[i]])的，那么我们可以每一轮从尾到头(cap...0)依次进行计算，
  2. 这样保证了每次dp[i][j]的整行的候选值都是上一轮dp[i-1][]的候选值，那么数组空间直接去掉行
  优化的算法如下：

public class Backpack {
    public static int maxValue(int[] w, int[] v, int n, int cap) {
        // write code here

        int[] dp = new int[cap + 1];

        dp[0] = 0;
        for (int i = 0; i < n; i++) {
            for (int j = cap; j >= w[i]; j--) {
                dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i]);
            }
        }

        return dp[cap];
    }
}

总结：1. 动态规划问题基本上就是对于暴力递归方法进行时间复杂度的优化，采取空间换时间的思路，但是每一步都按照指定逻辑严格执行。2.动态规划的初始化以及优化也是比较基础但是很重要的一点，每次使用动态规划时都需要注意。

1 传统的快速排序

1.1 基本概念

前言：因为快速排序是排序算法中最为重要，泛用性极高的算法。而且变体和改进还特别多，在此我就单独成章，把它拿出来说一说。

定义：快速排序是对冒泡排序的改进。因为他们都是通过扩大（升序排序中）数组的一半确定小于（冒泡排序是大于）另一半，来完成排序的。

思想：通过一趟排序实现将记录分割成独立的两部分，其中一部分记录的关键字均比另一部分记录的关键字小。则可分别对两部分记录进行排序，已达到整体有序的。

复杂度分析：
最优情况：
时间复杂度：最优情况下，每次都很均匀地分成两组，第一次划分比较的次数，每一组是n/2，那么两组就是n次，而递归地次数是O([long2n]+ 1)，设时间复杂度函数为T(x)，可得推导式：
T（n）≤2T（n/2） +n，T（1）=0
T（n）≤2（2T（n/4）+n/2） +n=4T（n/4）+2n
T（n）≤4（2T（n/8）+n/4） +2n=8T（n/8）+3n
……
T（n）≤n（2T（1） + n）+（log2n - 1）×n= O(nlogn)
所以时间复杂度为O(nlog2n).

空间复杂度：
每次递归都需要1个额外空间，所以空间复杂度为O(log2n)。

最坏情况：
时间复杂度：最坏情况下，序列是反序的，每次分组都是比上一次少划分一个记录的子序列，另一边则为空，这样就需要进行n-1次分组，递归树深度为n-1，每次比较了n-i次，一共比较了n*(n-1)/2次，所以时间复杂度为O(n^2)

稳定性：因为不是相邻交换了，所以快速排序是不稳定的。

评价：快速排序，大家说好，就是真的好。

1.2 代码实现

代码实现：

  • 找到枢纽位置pivot是关键步骤，取低位元素作为枢纽值，进行循环分组。
  • 每一趟分组，先从高位进行分组，选出比枢纽值小的值，与枢纽原位置进行交换（枢纽值处于高位）；接着低位开始进行分组，选出比枢纽值大的元素，与枢纽值进行交换（枢纽值处于低位），一趟结束。
  • 若低位仍然小于高位，就继续循环，直到低位与高位相等，循环结束

 * 快速排序。分治法的应用，先将数组根据第一个元素大小分成两边。

/**
 * 分组。根据第一个元素大小分成两边的数组，一边小，一边大。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 划分好的数组
 */
private static int getPartition(int[] array, int low, int high) {
    int pivot_key = array[low];

    // 从表的两端交替向中间扫描
    while (low < high) {
        while (low < high && array[high] >= pivot_key) {
            high--;
        }

        swap(array, low, high);

        while (low < high && array[low] <= pivot_key) {
            low++;
        }

        swap(array, low, high);
    }

    return low;
}

private static void swap(int[] array, int i, int j) {
    if (i != j) {
        int temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }
}

2. 改进的快速排序

2.1 快速排序的问题

问题:最直观的快排的问题出在它的递归实现上面，一旦数据量上去之后，而枢纽值选取不合理的话，很有可能出现线性次数的递归，然后产生爆栈（栈溢出异常）。次要问题是，在数据量太小的时候又性能一般，不如插入排序等等这些基础排序算法。

以上这些内容细分可以分为四个问题，我在这里一个一个解决：

1) 枢轴值选取不合理。
 问：为什么在正序或者反序的时候，快速排序的时间复杂度为平方阶O(n^2)？
 答： 因为对于这两种情况，一边的数组是空的，另一边又是待排序数组的完整版。而且分完组的话枢纽值要么最终在头上，要么最终在尾巴上，比较次数和比较的循环轮数都是最大的。最后的时间复杂度达到了O(n^2)。

2) 递归交换的复杂度较高
 问：这里面的分组函数getPartition每次调用swap都会进行3次操作，能不能改成替换，只进行一次操作？
 答：可以。

3) 性能问题（小数据样本时）
 问：当数组数量低于7个（资料显示）的时候，采用插入排序反而效率较高？
 答：博士生去小学教“1+1=2”也不见得比本科生教得好。快速排序对于基本有序，或者样本极少的数组排序的确效率不高。插入排序反而比较适合基本有序的小样本排序。

4) 递归占用太多栈空间
 问：大家知道，递归对性能是有一定影响的，quick_sort函数在其尾部有两次递归操作。如果待排序的序列划分极端不平衡（正序或反序），递归深度将趋近于n，而不是平衡时的log2n，栈的空间是极其有限的，如何减少递归？
 答：解决递归地最好方法就是使用迭代。所以我们需要优化这里面不必要的递归交换。甚至直接采用非递归的方法实现快速排序。

/**
 * 划分方法。完全按照左老师的方法来。
 * 1.交换枢划分值和数组高位（尾部）值，让划分值成为哨兵
 * 2.先在规定区域(high, low)的低位前一位规定一个小于等于划分值区域，初始长度为0
 * 3.移动index指针，遇到满足条件的数，就放在小于等于划分值区域的下一位（就是交换），区域长度          增加1，直到指针走到高位
 *
 * 虽然较好的去划分值的方法是三数取中，但是随机数也是可以的。
 * @param A 待排序数组
 * @param low 低位指针
 * @param high 高位指针
 * @return 划分值在数组中位置以及一个划分过的数组
 */
private static int getPartition(int[] array, int low, int high) {
    // 1.交换枢划分值和数组高位（尾部）值，让划分值成为哨兵
    swap(array, low, high);
    // 2.先在规定区域(high, low)的低位前一位规定一个小于等于划分值区域，初始长度为0
    int smallerTail = low - 1;
    int current = low;
    // 3.移动index指针，遇到满足条件的数，就放在小于等于划分值区域的下一位（就是交换），区域         // 长度增加1，直到指针走到高位
    while (current <= high) {
        if (array[current] <= array[high]) {
            swap(array, ++smallerTail, current);
        }
        current++;
    }
    return smallerTail;
}

 * 非递归实现，使用一个栈，用每次划分区域得出的中枢值将区域进行再次划分，
 * 直到中枢值等于区域的头或者尾部为止
 * @param array 待排序数组
 * @param head 数组头部
 * @param tail 数组尾部
 * @return 一个排好序的升序数组
 */
public static int[] quick_sort2(int[] array, int head, int tail) {
    int pivot;
    // 用一个栈来实现，每次划分只排一个数的序，就是pivot中枢值，当局部头尾等于pivot，
    // 就将其出栈，当得出的中枢 值和局部的头尾值不相等时，就将不等部分入栈，
    // 直到全部数字都出栈为止
    Stack<Integer> stack = new Stack<>();
    pivot = getPartition(array, head, tail);

    if (head < pivot - 1) {
        stack.push(head);
        stack.push(pivot);
    }
    if (tail > pivot + 1) {
        stack.push(pivot + 1);
        stack.push(tail);
    }

    while (!stack.isEmpty()) {
        int high = stack.pop();
        int low = stack.pop();

        pivot = getPartition(array, low, high);
        if (low < pivot - 1) {
            stack.push(low);
            stack.push(pivot - 1);
        }
        if (high > pivot + 1) {
            stack.push(pivot + 1);
            stack.push(high);
        }
    }

    return array;
}

    ListNode largerTail = largerHead;

    ListNode cur = head;
    while (cur != null) {
        if (cur.val > val) {
            largerTail.next = cur;
            largerTail = largerTail.next;
            // 处理小于值的链表，让小于指针next始终保持在链表current一致
            smallerTail.next = cur.next;
        } else {
            // 这里有奇效，只需要移动过去就可以了，且最后一位会自动置空
            smallerTail = cur;
        }
        cur = cur.next;
    }

    // 大于值的链表需要置空
    largerTail.next = null;
    smallerTail.next = largerHead.next;
    return smallerHead.next;
}

比较排序算法

1.前言

  根据排序过程中待排序记录是否全部存放在内存中，可以将排序算法归为两类：内排序和外排序。一般的记录数较少的情况下，采用的方法都是内排序；而记录数量巨大，而且要求排序算法效率高的情况下，采用的是外排序。

常见的内外排序有哪些？
内排序：插入排序算法、选择排序算法、交换排序算法、归并排序算法
外排序：外归并排序算法

比较排序算法有哪些？性能表现又是如何？

注1：关于 Memory，如果算法为 “in place”（原地） 排序，则仅需要 O(1) 内存；有时对于额外的 O(log(n)) 内存也可以称为 “in place”（原地）。

注2：Microsoft .Net Framework中Array.Sort方法就是实现了内省排序算法。

2.内排序

2.1插入排序 （Insertion Sorts）

2.1.1 直接插入排序 (Directly Insertion Sort)

定义：将一个记录插入一个有序表中，从而得到一个新的，记录数加一的有序表。

思路：无序插入有序表，每次从位置K由后向前进行比较，找到第一个不大于（小于）A[K]的数字，否则就放在数组头部。

复杂度分析：

  • 最好情况：正序情况，正好每1个内层循环都只比较了一次，一共比较了(n-1)次，没有交换
  • 最差情况：逆序情况，正好每个内层循环比较了等差数列次，第一次比较了1次，交换了1次，第二次比
    较了2次，交换了2次···第n-2次循环比较了(n-1)次，交换了(n-1)次；第(n-1)次循环比较了(n-1)次， 交换了(n-1)次。共比较 1 + 2 + ··· + (n-2) + (n-1) = n * (n-1) / 2次，交换n(n-1)/2次
  • 平均情况：O(n^2)

稳定性：连续比较，相同不交换，稳定

评价：对于基本有序的序列，插入排序的复杂度较低，但是插入排序的效率较低，每次只移动了一位

代码实现：采用双层循环，外层循环每次将数据跟前面的有序表进行对比，如果小于，就交换数字，往前对比；若不小于，则停止前进。
注释：这里的话作为交换条件的temp和array[j]的比较加到到循环里面去合理一些，所以说内层循环使用while更加简洁。

代码：

/**
 * 直接插入排序
 * @param array 待排序数组
 * @return 排序完成数组
 */
public static int[] directly_insert_sort(int[] array) {
    int length = array.length;
    for (int i = 1; i < length; i++) {
        // 使用一个额外空间来存放一个临时变量
        int temp = array[i];
        int j = i - 1;
        // 进行连续比较，直到数组开头
        while (j >= 0 && temp < array[j]) {
            array[j + 1] = array[j];
            j--;
        }
        array[j + 1] = temp;
    }
    return array;
}

2.1.2 希尔排序 (Shell Sort)

定义：是直接插入的改进，分组插入排序。希尔排序利用了插入排序对有序数组排序性能大增的特点，实现了计算

思路：通过选择特定的步长dl，将比较的元素分成几个区域来（进行插入排序）提升插入排序的性能。每次将区域缩小一点，最后缩小到步长dl=1的时候，就是普通的插入排序了。

复杂度分析：
步长不同，所得的时间复杂度也不同：

步长序列 最坏情况下复杂度
n/2^i O(n^2)
2^k - 1 O(n^{3/2})
2^i*3^j O( n*log^2 n)

时间复杂度：
这里采用的dlta(i) = n/2^i，时间复杂度是O(n^2)，若采用其他步长序列，则会更加高效。
平均时间复杂度为O(n^2)。
dlta(i) = 2^k-1，平均价时间复杂度为O(n^{3/2})，低于O(n^2)，这就是Shell排序划时代的地方了。

稳定性：跨越比较，相等元素之间可能交换，不稳定。

评价：希尔排序是最早打破人们思维定式，终结O(n^2)时间复杂度，突破慢速排序时间的排序算法。

代码实现：

  • 采用分割记录法，将数组分成(n/dl)个子序列
  • 对子序列进行直接插入排序
  • 步长的选择是排序关键。只要最终步长为1的所有步长序列都可以最终完成排序。算法发明者Shell推荐的步长序列是n/2，对步长取半直到步长为1

希尔排序最重要特性：对数组进行较小步长排序之后，之前用的较大步长仍然是有序的。

代码：

/**
 * 希尔排序。步长为n/2 
 * @param array 待排序数组
 * @return 排序完成数组
 */
public static int[] shell_sort(int[] array) {
    int len = array.length;
    int i;
    int j;
    int gap = len / 2;

    // 选择每一次的步长（外一层循环）；按照步长分出子序列；
    // 对每一组进行直接插入排序（内两层循环）
    for (;gap > 0; gap = gap / 2) {
        for (i = gap; i < len; i++) {
            int temp = array[i];
            for (j = i - gap; j >= 0 && array[j] > temp; j -= gap) {
                array[j + gap] = array[j];
            }
            array[j + gap]  = temp;
        }
    }
    return array;
}

2.2 交换排序 (Exchange Sort)

2.2.1 普通冒泡排序 (Bubble Sort)

定义： 对普通交换排序的改进，对无序区的记录依次进行比较，使得最大（小）的记录如同“气泡”一样“冒出水面”。

思想：通过对无序区记录进行两两比较，若反序就进行交换，一直交换到序列尾部，使元素成为有序区记录。依次交换，直到没有反序的记录为止。

复杂度分析：这里要对它进行改进，之后再对其时间复杂度进行分析。

稳定性：因为是相邻交换，所以冒泡排序是稳定排序。

代码实现：通过两层循环，从序列头部开始进行比较，反序就交换，从下往上交换。注意：尾部指的是无序区的尾部。

代码：

/**
 * 冒泡排序
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] bubble_sort(int[] array) {
    int length = array.length;
    for (int i = 0; i < length - 1; i++) {
        for (int j = 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
            }
        }
    }
    return array;
}

private static void swap(int[] array, int i, int j) {
    int temp = array[i];
    array[i] = array[j];
    array[j] = temp;
}

2.2.2 优化的冒泡排序

思想：普通冒泡可能会产生许多无用的比较，尤其是在顺序情况下，强行比较；为了避免这种情况，可以记录下上一轮的比较是否进行了交换，有交换继续排序，没有则是已经有序，结束排序。

复杂度分析：

  • 时间复杂度：
    最好情况：正序情况下，比较(n-1)次，交换0次，时间复杂度O(n)
    最坏情况：反序情况下，比较n*(n-1)/2次，交换同样多的次数，时间复杂度为O(n^2)
    平均情况：时间复杂度O(n^2)

  • 空间复杂度：原地排序，辅助空间为O(1)，交换的时候用到了辅助空间

评价：优化后的交换排序思路简单，对小样本性能也较好，是好用的排序。

代码实现：设置一个标记位FLAG，每次外层循环设为FALSE，在内层循环中，若有一次交换就设置为TRUE。

代码：

/**
 * 改进的冒泡排序。可以避免冗余的比较和交换
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] modified_bubble_sort(int[] array) {
    int length = array.length;
    boolean flag = true;

    for (int i = 0; i < length - 1 && flag; i++) {
        flag = false;
        for (int j = 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
                flag = true;
            }
        }
    }
    return array;
}

2.2.3 鸡尾酒排序（Cocktail Sort）

思想：鸡尾酒排序，也叫双向冒泡排序(Bidirectional Bubble Sort)，这一下就很清晰了。以升序排列为例，
它先把最大的数移到序列末尾，然后接着从还没排好序的序列从尾部开始，把最小的数字移到最前边开头的地方。来回这样序列长度的一半的次数，排序就完成了

算法复杂度
时间复杂度：最好O(n)，最差O(n^2)，平均O(n^2)

空间复杂度：最差O(1)

代码：

/**
 * 双向冒泡排序
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] cocktailSort(int[] array) {
    int length = array.length;
    for (int i = 0; i < (length - 1) / 2; i++) {
        // 先把最大的数移到后边
        for (int j = i + 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
            }
        }

        // 再把最小的数移到最前边
        for (int k = length - i - 1; k > i; k--) {
            if (array[k] < array[k - 1]) {
                swap(array, k, k - 1);
            }
        }
    }
    return array;
}

2.2.4 奇偶排序（Odd-Even Sort）

思想：选取奇数或者偶数为起始位置，以2位比较长度，两两进行比较。

代码实现：比较从第一位开始，下标能被2整除就是奇数排序，否则就是偶数排序，然后比较每个起始位置和下一个位置的大小，每次排序的增量为2，直到到达最后一位。

代码：

/**

 * 奇偶排序
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] oddEvenSort(int[] array) {
    int length = array.length;
    for (int i = 0; i < length; i++) {
        // 先判断是否能被2整除
        if (i % 2 != 0) {
            // 进行偶数排序
            for (int j = 2; j < length; j += 2) {
                if (array[j] < array[j - 1]) {
                    swap(array, j, j - 1);
                }
            }
        } else {
            // 进行奇数排序
            for (int j = 1; j < length; j += 2) {
                if (array[j] < array[j - 1]) {
                    swap(array, j, j - 1);
                }
            }
        }

    }
    return array;
}

2.2.5 快速排序（Quick Sort） — 最重要的排序算法

定义：对冒泡排序的改进。

思想：通过一趟排序实现对记录的分割，其中一部分记录的关键字均比另一部分记录的关键字小。则可分别对两部分记录进行排序，已达到整体有序的。

2.3 选择排序

2.3.1 简单选择排序

思想：每一趟从未排序元素中选择出一个最小（最大）的值，放在队头。

复杂度分析：
时间复杂度：最好，最坏比较都是(N-1)*(N+1)/2,交换是N-1次，所以时间复杂度是O(n^2)
空间复杂度：原地排序，一个临时变量，所以是常数级别的O(1)
稳定性:因为是相邻元素的比较，而且中途没有交换，所以是稳定的，而且是特别稳定，任何情况的比较和交换（不算赋值过程）都是一样的

评价：交换的次数较少，但是比较的次数较多，普通性能的排序。

代码实现：使用两层循环，每次用min（max）保存一个最值，对未排序元素进行依次比较，得出的结果与未排序元素头元素进行交换，然后又指针（数组下标）往后移动一位，形成新的无序元素表，循环直到剩余元素为空。

代码：

/**
 * 简单选择排序，第i轮循环记录一个最小值的位置，在循环结束后将位置为i的元素与最小值交换
 * @param array 待排序序列
 * @return 排好序序列
 */
public static int[] simple_select_sort(int[] array) {
    int length = array.length;
    int pos;

    for (int i = 0; i < length - 1; i++) {
        pos = i;
        for (int j = i + 1; j <= length - 1;j++) {
            if (array[j] < array[pos]) {
                pos = j;
            }
        }
        // 把最小元素换到队头来
        swap(array, i, pos);
    }
    return array;
}

private static void swap(int[] array, int i, int j) {
    if (i != j) {
        int temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }
}

稳定性：

评价：堆排序是性能较好的排序，只是空间占用会大一点，不好理解一点。

2.5.1 原始的归并排序

预备知识：

  1. 归并在数据结构中的定义是：将两个或者两个以上的有序表组合成一个新的有序表。
  2. 归并算法不唯一：多路归并算法有多种实现它的路数是浮动的K，我们在这里讲的是最简单的二路归并。
  3. 分治法：将一个复杂的问题不断地分成两个或者更多相同或相似的子问题来解决，直到最后子问题可以直接简单求解为止，子问题的解得合并即为原问题的解。

思想：二路归并算法是利用分治法的思想。它假设初始序列有N个记录，将这N个元素看做N个有序子序列，每个子序列的长度为1，然后进行合并。

复杂度：
时间复杂度：首先， 归并排序需要将temp[0]…temp[n]中相邻长度为h的所有记录两两排序归，并存放到target[0]…target[n]里面去。此过程，需要将temp所有记录都扫描一遍，因此耗费O(n)的时间，而由完全二叉树的深度为[log2(n)]+1可知，归并排序需要进行[log2(n)]的常数倍次，因此，总的时间复杂度为O(nlog2(n))，而且是最好、平均、最坏都是如此。

评价：归并排序在外排序的时候用的更多一些，跟堆排序一样的缺点。

 * @param array 待排序序列

public static int[] merge_sort(int[] array) {
    int length = array.length;
    if (length <= 1) {
        return array;
    }
    length = array.length;
    int[] target = new int[length];

    return m_sort(array, target, 0, length - 1, length);
}

/**
 * 归并排序算法
 * @param array 待排序序列
 * @param target 排好序输出的目标序列
 * @param start 排序起始位置
 * @param end 排序末尾位置
 * @param length 待排序序列长度
 * @return 排好序序列
 */
private static int[] m_sort(int[] array, int[] target, int start, int end, int length) {
    int middle;
    int[] temp = new int[length];

    if (start == end) {
        target[start] = array[start];
    } else {
        middle = (start + end) / 2;

        m_sort(array, temp, start, middle, length);
        m_sort(array, temp, middle + 1, end, length);
        // 两两归并
        merge(temp, target, start, middle, end);
    }

    return target;
}

/**
 * 将数组temp[start...middle]和temp[middle+1...end]归并起来
 * 其实就是讲两个序列有序地合并，先在两个序列都不为空的情况下进行元素比较，
 * 哪个小插入哪个到尾部，直到其中至少一个序列为空，然后将剩余的序列依次插入尾部
 * 这里面的两个序列就来自于数组temp的前半（start...middle）和后半（middle+1...end）
 *
 * @param temp 待排序总序列
 * @param target 排好序之后输出的序列
 * @param start 前半序列起始位置
 * @param middle 中间分隔两个序列的位置，前半序列的中点位置，后半序列起始位置的前一个位置
 * @param end 后半序列的末尾位置
 */
private static void merge(int[] temp, int[] target, int start, int middle, int end) {
    int i = start;
    int j = middle + 1;
    int k = start;// target指针

    // 有序地合并序列，特别注意，这里target的插入起始位置跟temp的起始位置是一样的
    while (i <= middle && j <= end) {
        if (temp[i] < temp[j]) {
            target[k++] = temp[i++];
        } else {
            target[k++] = temp[j++];
        }
    }

    // 若前半没合并完，依次插入尾部
    while (i <= middle) {
        target[k++] = temp[i++];
    }

    // 若后半没合并完，依次插入尾部
    while (j <= end) {
        target[k++] = temp[j++];
    }
}

2.5.2 改进的归并排序

问题的提出：因为空间耗费大是归并排序的一大缺陷，而且这一缺陷是递归造成的。所以在这里，可以将递归优化成迭代。减少空间占用。

思想：将递归换成迭代，利用新创建的target数组与源数组辗转排序最终完成排序。

代码：

/**
 * 归并排序算法。从最小的间距为2开始，对数组进行迭代排序
 * @param array 待排序序列
 * @return 排好序序列
 */
private static int[] modified_m_sort(int[] array) {
    int interval = 1;
    int length = array.length;
    int[] target = new int[length];

    // 之所以调用两次。1.非原地排序，需把两个序列来回倒换
    // 2.即使interval长度超过了序列总厂，也有判断可以避免错误indexOutOfRange
    while (interval < length) {
        mergePass(array, target, interval, length);
        interval *= 2;

        if (interval > length) {
            return target;
        } else {
            mergePass(target, array, interval, length);
            interval *= 2;
        }

    }

    return array;
}

/**
 * 通过对序列里面的子序列进行排序进行，从头到尾，将标准长度的子序列两两归并
 * @param srcArray 待排序序列
 * @param targetArray 排好序序列
 * @param interval 每个归并子序列标准长度
 * @param length 待排序序列总长度
 */
private static void mergePass(int[] srcArray, int[] targetArray, int interval, int length) {
    int i = 0;
    int j;
    // 以两个间隔为单位，进行归并排序
    while (i <= length - 1 - (2 * interval - 1)) {
        merge(srcArray, targetArray, i, i + interval - 1, i + 2 * interval - 1);

        i += 2 * interval;
    }

    // 还剩不止一个子序列，进行两两归并
    if (length - 1 - i > interval - 1) {
        merge(srcArray, targetArray, i, i + interval - 1, length - 1);
    } else {
        // 最后若只剩下单个子序列，直接插入尾
        for (j = i; j < length; j++) {
            targetArray[j] = srcArray[j];
        }
    }
}

 * 其实就是讲两个序列有序地合并，先在两个序列都不为空的情况下进行元素比较，
 * 哪个小插入哪个到尾部，直到其中至少一个序列为空，然后将剩余的序列依次插入尾部
 * 这里面的两个序列就来自于数组temp的前半（start...middle）和后半（middle+1...end）
 *
 * @param temp 待排序总序列
 * @param target 排好序之后输出的序列
 * @param start 前半序列起始位置
 * @param middle 中间分隔两个序列的位置，前半序列的中点位置，后半序列起始位置的前一个位置
 * @param end 后半序列的末尾位置
 */
private static void merge(int[] temp, int[] target, int start, int middle, int end) {
    int i = start;
    int j = middle + 1;
    int k = start;// target指针

    // 有序地合并序列，特别注意，这里target的插入起始位置跟temp的起始位置是一样的
    while (i <= middle && j <= end) {
        if (temp[i] < temp[j]) {
            target[k++] = temp[i++];
        } else {
            target[k++] = temp[j++];
        }
    }

    // 若前半没合并完，依次插入尾部
    while (i <= middle) {
        target[k++] = temp[i++];
    }

    // 若后半没合并完，依次插入尾部
    while (j <= end) {
        target[k++] = temp[j++];
    }
}

因为没有使用递归，所以这里的空间复杂度降低了，但是在interval最大的一次归并还是需要扫描整个序列的一半以上，所以空间复杂度仍然是O(n)，但是相比于递归已经好很多了。
时间复杂度：因为i是以2为底的指数，所以时间复杂度和改进之前的一样，都是O(nlogn)。所以，我们一半使用的归并排序都是非递归形式的归并排序。

预定义：这里有一个预定义，数组里面的所有值都是自然数，也就是非负整数。
思想：找到待排序数组中所有数值的范围，根据范围创建一个计数数组，范围(min, max)。

复杂度分析：

空间复杂度：计数数组为O(K)，结果数组O(N)，总空间复杂度为O(N + k)

private static void countingSort(int[] A, int[] B, int k) {
    int[] C = new int[k];
    // 计数
    for (int j = 0; j < A.length; j++) {
        int a = A[j];
        C[a] += 1;
    }

    // 求计数和，此计数和就是数字在有序数组中的（位置+1）
    for (int i = 1; i < k; i++) {
        C[i] = C[i] + C[i - 1];
    }

    // 整理
    for (int j = A.length - 1; j >= 0; j--) {
        int a = A[j];
        B[C[a] - 1] = a;
        C[a] -= 1;
    }
}

优化计数数组版本代码：

 /**
 * 计数排序。
 * 先得出待排序数组中最大的数字，根据数字将放进计数数组。
 * 然后将计数数组依次累加得到数字当前的位置
 * 最后从待排序数组尾部开始读数字，放入结果数组。
 * 针对数组大小，优化过的计数排序
 * @param A 待排序数组
 * @param n 待排序数组长度
 * @return 排好序数组
 */
public static int[] countingSort(int[] A, int n) {
    if (n < 2) {
        return A;
    }

    // 0.计数数组的长度
    int max = A[0];
    int min = A[0];
    int length;

    for (int i = 1; i < A.length; i++) {
        if (max < A[i]) {
            max = A[i];
        }
    }

    for (int i = 1; i < A.length; i++) {
        if (min > A[i]) {
            min = A[i];
        }
    }

    length = max - min + 1;
    int[] countingArray = new int[length];
    int[] orderedArray = new int[n];

    int number;
    // 1.计数。先把每个数字的频率记录下来，放入计数数组
    for (int i = 0; i < n; i++) {
        number = A[i];
        countingArray[number - min]++;
    }

    // 2.求计数和。再把每个数字的频率累加，放入计数数组，就是当前数字的位置
    for (int j = 1; j < length; j++) {
        countingArray[j] = countingArray[j] + countingArray[j - 1];
    }

    // 3.从最后一位开始，放入结果数组，随着待排序数组数字放入结果数组，计数数组的放入位置应当减1
    for (int k = n - 1; k >= 0; k--) {
        number = A[k];
        // 数组实际位置和个数直接相差1，所以要减1
        orderedArray[--countingArray[number - min]] = number;
    }

    return orderedArray;
}

 * 基数排序，就是计数排序的多位数版本。桶排序的常见实现。
 * 有MSD和LSD两种实现，我们这里写的是最为通用的LSD。
 * 1.先判断数组最大数字的位数
 * 2.根据位数，进行从低位到高位的循环
 * @param A 待排序数组
 * @param n 待排序数组长度
 * @return 排好序数组
 */
public static int[] radixSort(int[] A, int n) {
    if (n < 2) {
        return A;
    }

    // 1.先找出最大位数
    int maxBit = findMaxBit(A, n);
    int radix = 1;
    int i;
    int j;

    // 临时数组和计数数组，计数数组是十进制的
    int[] temp = new int[n];
    int[] count = new int[10];

    // 2.根据位数，进行从低位到高位的循环
    for (i = 1; i <= maxBit; i++) {
        // 将技术数组重新初始化为0
        for (j = 0; j < 10; j++) {
            count[j] = 0;
        }
        // 将待排序数的尾缀数字0-9出现频率放进计数数组
        for (j = 0; j < n; j++) {
            int k = (A[j] / radix) % 10;
            count[k]++;
        }

        // 求计数和，也就是所有数字的理论位置
        for (j = 1; j < 10; j++) {
            count[j] = count[j - 1] + count[j];
        }

        // 根据数字的实际位置来存放到临时数组
        for (j = n - 1; j >= 0; j--) {
            int k = (A[j] / radix) % 10;
            temp[--count[k]] = A[j];
        }

        // 将排好序的数字返回给原数组
        for (j = 0; j < n; j++) {
            A[j] = temp[j];
        }

        // 进位
        radix *= 10;
    }

    return A;
}

private static int findMaxBit(int[] A, int n) {
    int max = A[0];
    int d = 1;
    // 先找最大值
    for (int i = 0; i < n; i++) {
        if (A[i] > max) {
            max = A[i];
        }
    }

    while (max >= 10) {
        max /= 10;
        d++;
    }

    return d;
}

堆排序

因为堆排序的时空复杂度是比较特殊和有趣的，所以这里单拎出来说一说。

  • 堆排序的空间复杂度O(1)，因为只使用了一个辅助空间。

  • 堆排序的时间复杂度，建堆时间O(N)，调整堆时间O(Nlog2(N))。对于堆排序的时间复杂度。可以分为两个部分分别讨论。

  1. 调堆时间，因为一共N个元素所以树的高度H=log2(N)+1（性质4），每次是从根节点开始比较，比较最多比较了2(H-1)次，交换的最坏情况调整了(H-1)，那么一共有N-1个元素需要调整，可以估算时间复杂度就是O(Nlog2(N))。
  2. 关键是建堆时间，因为是从最后一个非叶子结点开始调整，所以肯定调整了与N同量级的次数，时间复杂度不可能是log2(N)(H)。但是会不会是O(Nlog2(N))？因为每次最后一个最后一个非叶子结点入堆顶都得使用时间为(log2(N)-1)的时间，而非叶子结点的个数为N/2，那么O(Nlog2(N))看起来很合理不是吗？
    实际上不是的，因为不是从根开始比较而是从第一个非叶子结点开始比较，比较次数少了很多。具体计算，我们从底层开始算起，最后一层（H层）父节点一共有2^(H-1)个结点，他们最多需要交换的次数是1次，比较的次数是2次；倒数第二层的一共有2^(H-2)个结点，他们最多需要交换的次数是2次，比较的次数是4次，倒数第二层父节点的个数是2个，他们最多需要交换的次数是H-2次，比较的次数是2(H-2)次；根节点就1个，它最多需要交换的次数是(H-1)次，比较的次数是2(H-1)次。

所以说，最坏情况下的交换次数是S = 1 2^(H-1) + 2 2^(H-2) + ··· + (H-2) 2^1 + (H-1) 2^0 = 2N - 2 - log2(N),最坏情况下比较的次数是 2S 次，所以时间复杂度是O(3S = 6N - 6 - 3log2(N))，即为O(N);
没有排序是完美的，堆排序尽管非常优秀，有很高效的时间复杂度（平均O(nlogn)），而且有很低的空间复杂度O(1)，但是由于非连续比较的缘故，并不是稳定的排序算法。

附加专题：完全二叉树的5个性质推导

  1. 所有二叉树的第i层上最多拥有2^(i-1)个节点。
    证明：数学归纳法
    1层 1个 2^0
    2层 2个 2^1
    …
    假设 k层 2^k个
    k+1 层 2*2^(k+1) = 2^(k+1)个【父节点下最多拥有两个子节点】
    …
    n层 2^n个
    所以，性质成立。
  2. 二叉树中如果深度为k,那么最多有2^k - 1个节点
    证明：根据性质1，求k层满二叉树的节点个数，个数s = (1+2+…+2^(k-1)) = 1 * (1 - 2^k)/ (1 - 2) = 2^k - 1。
    注释：满二叉树就是每一层都是满节点的特殊二叉树，是完全二叉树的究极形态。
  3. n0=n2+1 n0表示度数为0的节点 n2表示度数为2的节点
    证明：根据两个公式：
    n = n0 + n1 + n2
    n - 1 = n1 + 2 n2 入度等于出度，入度是除了根节点外其他节点都为1，出度是按定义计算。
    两个公式通过n作为桥梁，就可以化简为 (n0 + n1 + n2) - 1 = n1 + 2     n2
    得到n0=n2+1
    1. 在完全二叉树中，具有n个节点的完全二叉树的高度是[log2(n)] + 1，其中，[log2(n)] 是向下取整。
      证明：我们假设这个完全二叉树的高度为k.
      对一个高度为k的完全二叉树而言，它的节点数一定比高度为(k-1)的满二叉树要多（大于），小于等于高度为k的满二叉树。根据性质2，可以得不等式 2^(k-1) - 1 < n <= 2^k - 1。因为n为整数，
      所以不等式可以大胆地加减1，得：2^(k-1) -1 + 1 <= n < 2^k - 1 + 1 。
      对不等式去以二为底的对数，得： k - 1 <= log2(n) < k。所以 k = [log2(n)] + 1，其中，[log2(n)] 是向下取整。
    2. 如果有一颗有n个节点的完全二叉树的节点按层次序编号，对任一层的节点i（1<=i<=n）有：
    • 如果i = 1，则节点为根节点，无双亲，如果i > 1， 那么它的双亲为[i/2]，向下取整
    • 如果2i > n，节点i没有左孩子节点，否则左孩子节点为2i
    • 如果2i + 1 > n，节点i没有右孩子节点，否则右孩子节点为2i + 1
    • 最后一个非叶子节点编号为 n/2

第五条：对Java而言，数组从0开始，那么规律就需要变化一下

  • 如果i = 0，则节点为根节点，无双亲，如果i > 0， 那么它的双亲为[(i - 1)/2]，向下取整
  • 如果2i + 1 > n，节点i没有左孩子节点，否则左孩子节点为2i + 1
  • 如果2i + 2 > n，节点i没有右孩子节点，否则右孩子节点为2i + 2
  • 最后一个非叶子节点编号为 n/2 - 1

public static String allcateSubnet(String ipPool, Integer subnetMask) {
    // 将地址池分为ip和掩码，分别进行处理
    String[] subnet = ipPool.split("\\/");
    // 地址池的上限，如果达到次上限，子网就算分完了
    String ipPoolLimit;
    // 分配好的子网的IP
    String allocatedIp;
    // 分配好的子网
    String allocatedSubnet = null;
    // ip地址池的二进制字符串
    String binaryIpPool;
    // 地址池的掩码部分
    Integer ipPoolMask = 0;
    // 分配成功的标志
    boolean flag = false;

    // 第一步，判断subnetMask是否合法
    if (subnet[1] != null) {
        try {
            ipPoolMask = Integer.valueOf(subnet[1]);
        } catch (Exception e) {
            System.out.println("mask Exception : " + e.getMessage());
            return null;
        }
    }

    if (ipPoolMask >= subnetMask) {
        System.out.println("subnetMask should be less than ip pool mask.");
        return null;
    }

    // 第二步，将地址池的ip部分转化成二进制数，并且将其与地址池的上限进行比较
    try {
         binaryIpPool = ipToBinary(subnet[0]);
    } catch (Exception e) {
        System.out.println("Ip string illegal.Exception : " + e.getMessage());
        return null;
    }

    // 第三步，得到地址池的上限，并且对第一次分配子网进行初始化
    ipPoolLimit = binaryAdd(binaryIpPool, ipPoolMask);

    if (currentGreatestIp == null) {
        currentGreatestIp = binaryIpPool;
        allocatedIp = currentGreatestIp;
    } else {
        allocatedIp = binaryAdd(currentGreatestIp, subnetMask);
    }

    // 第四步，IP的最终分配。满足条件：
    // 1. 分配的IP小于上限IP
    // 2. 分配的IP未被覆盖到
    // 3. 还有剩余子网

    // 第一次给地址池分IP，先分了再说
    if (subnetIpMap.get(ipPool) == null) {
        List<String> ipList = new LinkedList<>();
        ipList.add(allocatedIp);
        subnetIpMap.put(ipPool, ipList);
    }
    if (ipPoolUsableMap.get(ipPool) == null) {
        ipPoolUsableMap.put(ipPool, true);
        return ipToSubnet(allocatedIp, subnetMask);
    }

    //地址池已经分完，不可再分
    if (!ipPoolUsableMap.get(ipPool)) {
        System.out.println("No subnet available.");
        return null;
    }

    // 第二次及以后分配ip, 没成功就一直分
    // 若在范围内就一直加
    if (!checkSubnetOverIpRange(allocatedIp, ipPoolLimit)) {
        while (!flag) {
            if (!checkSubnetIpIsInUsed(allocatedIp, ipPool)) {
                List<String> ipList = subnetIpMap.get(ipPool);
                ipList.add(allocatedIp);
                subnetIpMap.put(ipPool, ipList);
                currentGreatestIp = allocatedIp;
                allocatedSubnet = ipToSubnet(allocatedIp, subnetMask);

                flag = true;
            } else {
                allocatedIp = binaryAdd(allocatedIp, subnetMask);
            }
        }
    } else {
        // 若allocatedIp超过上限，就从自由的地址池里面分
        if (freeSubnetIpMap.get(ipPool) != null && freeSubnetIpMap.get(ipPool).size() != 0) {
            allocatedIp = binaryIpPool;

            List<String> freeIpList = freeSubnetIpMap.get(ipPool);

            while (!flag) {
                if (freeIpList.contains(allocatedIp)) {
                    freeIpList.remove(allocatedIp);
                    freeSubnetIpMap.put(ipPool, freeIpList);

                    List<String> ipList =  subnetIpMap.get(ipPool);
                    ipList.add(allocatedIp);
                    subnetIpMap.put(ipPool, ipList);
                    currentGreatestIp = allocatedIp;
                    allocatedSubnet = ipToSubnet(allocatedIp, subnetMask);

                    flag = true;
                } else {
                    allocatedIp = binaryAdd(allocatedIp, subnetMask);
                }
            }
        } else {
            // 自由子网也分配完了就是没有真的地址了
            ipPoolUsableMap.put(ipPool, false);
            System.out.println("No subnet available.");
            return null;
        }
    }

    return allocatedSubnet;
}

private static boolean checkSubnetIpIsInUsed(String allocatedIp, String ipPool) {
    List<String> ipList = subnetIpMap.get(ipPool);
    return ipList.contains(allocatedIp);
}

// 判断allocatedIp是否超越了上限
private static boolean checkSubnetOverIpRange(String allocatedIp, String ipPoolLimit) {
    // 获取allocatedIp的二进制字符串字段
    int[] allocateNum = new int[4];
    int[] limitNum = new int[4];

    for (int i = 0; i < allocateNum.length; i++) {
        try {
            allocateNum[i] = Integer.valueOf(allocatedIp.substring(i * 8, (i+1) * 8), 2);
            limitNum[i] = Integer.valueOf(ipPoolLimit.substring(i * 8, (i+1) * 8), 2);
        } catch (Exception e) {
            System.out.println("Ip string illegal.Exception : " + e.getMessage());
            return true;
        }
    }

    // 比较两个子网ip的大小,一旦allocatedIp大于等于ipPoolLimit，返回false
    if (allocateNum[0] > limitNum[0]) {
        return true;
    } else if (allocateNum[0] == limitNum[0]) {
        if (allocateNum[1] > limitNum[1]) {
            return true;
        } else if (allocateNum[1] == limitNum[1]) {
            if (allocateNum[2] > limitNum[2]) {
                return true;
            } else if (allocateNum[2] == limitNum[2]) {
                if (allocateNum[3] > limitNum[3]) {
                    return true;
                } else if (allocateNum[3] == limitNum[3]) {
                    return true;
                }
            }
        }
    }

    return false;
}


// 将二进制的ip地址和子网掩码的二进制数相加，得到的就是一个子网的ip
private static String binaryAdd(String binaryIp, Integer mask) {
    //先将掩码转换成二进制数
    Integer middle = 1;
    String binaryMask;
    String allocatedIp;

    for (int i = 0; i < 32 - mask; i++) {
        middle *= 2;
    }
    binaryMask = Integer.toBinaryString(middle);

    int[] ca = new int[binaryIp.length()];
    int[] cb = new int[binaryMask.length()];

    // 将字符数组转换成数值的0或者1
    for (int i = 0; i < binaryIp.length(); i++) {
        ca[i] = binaryIp.charAt(i) - '0';
    }
    for (int i = 0; i < binaryMask.length(); i++) {
        ca[i] = binaryMask.charAt(i) - '0';
    }

    // 用ca保存长度的长
    if (ca.length < cb.length) {
        int[] temp = ca;
        ca = cb;
        cb = temp;
    }

    // 字符数组中最后一个索引下标，也是最小的一个
    int la = ca.length - 1;
    int lb = cb.length - 1;

    // 下位的进位标识符
    int carry = 0;

    //两位相加的结果
    int result;

    while (la >= 0 && lb >= 0) {
        result = ca[la] + cb[lb] + carry;

        ca[la] = result % 2;
        carry = result / 2;

        la--;
        lb--;
    }

    // 处理余下的数字
    while (la >= 0) {
        result = ca[la] + carry;
        ca[la] = result % 2;
        carry = result / 2;

        if (la == 0) {
            break;
        }

        la--;
    }

    // 将计算结果换回字符
    char[] chars = new char[ca.length];
    for (int i = 0; i < ca.length; i++) {
        ca[i] += '0';
        chars[i] = (char)ca[i];
    }

    return String.valueOf(chars);
}

// 将十进制的字符串ip转化成二进制数
private static String ipToBinary(String ip) {
    String[] binaryIps = new String[4];

    // 先将ip切分成4个ip段
    if (ip != null) {
        String[] ips = ip.split("\\.");

        for (int i = 0; i < ips.length; i++) {
            try {
                binaryIps[i] = Integer.toBinaryString(Integer.valueOf(ips[i]));
            } catch (Exception e) {
                throw new IllegalArgumentException();
            }
        }
    } else {
        System.out.println("Ip string illegal!");
        return null;
    }

    // 将4个ip段分别转化成2进制再进行拼接，得到一个32位的数字
    StringBuilder stringBuilder = new StringBuilder(1024);
    for (String binaryIp : binaryIps) {
        String ipSegment;
        char[] ipChars = binaryIp.toCharArray();
        char[] binaryChars = new char[IP_SEGMENT_LENGTH];

        for (int j = 0; j < binaryChars.length; j++) {
            if (j < ipChars.length) {
                binaryChars[binaryChars.length - 1 -j] = ipChars[j];
            } else {
                binaryChars[binaryChars.length - 1 -j] = '0';
            }
        }
        ipSegment = String.valueOf(binaryChars);
        stringBuilder.append(ipSegment);
    }

    return stringBuilder.toString();
}

// 二进制IP转化成标准子网形式
private static String ipToSubnet(String allocatedIp, Integer subnetMask) {
    // 获取allocatedIp的二进制字符串字段
    Integer[] allocateNum = new Integer[4];
    StringBuilder sb = new StringBuilder(1024);

    for (int i = 0; i < allocateNum.length; i++) {
        try {
            allocateNum[i] = Integer.valueOf(allocatedIp.substring(i * 8, (i+1) * 8), 2);
        } catch (Exception e) {
            System.out.println("Ip string illegal.Exception : " + e.getMessage());
            return null;
        }
        if (i == allocateNum.length - 1) {
            sb.append(allocateNum[i]).append("/").append(subnetMask.toString());
            continue;
        }
        sb.append(allocateNum[i]).append(".");
    }

    return sb.toString();
    }
}
