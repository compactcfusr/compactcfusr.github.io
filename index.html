排序

今天的专题，我会从尽量概括和全面的角度来将自己在算法方法学习到的知识总结出来，提纲挈领。

#1.前言
根据排序过程中待排序记录是否全部存放在内存中，可以将排序算法归为两类：内排序和外排序。一般的记录数较少的情况下，采用的方法都是内排序；而记录数量巨大，而且要求排序算法效率高的情况下，采用的是外排序。

常见的内外排序有哪些？
内排序：插入排序算法、选择排序算法、交换排序算法、归并排序算法
外排序：外归并排序算法

#2.内排序

##2.1插入排序

###2.1.1 直接插入排序
定义：将一个记录插入一个有序表中，从而得到一个新的，记录数加一的有序表。

思路：无序插入有序表，每次从位置K由后向前进行比较，找到第一个不大于（小于）A[K]的数字，否则就放在数组头部。

代码实现：采用双层循环，外层循环每次将数据跟前面的有序表进行对比，如果小于，就交换数字，往前对比；若不小于，则停止前进。

代码：

/**
 * 直接插入排序
 * @param array 待排序数组
 * @return 排序完成数组
 */
public static int[] directly_insert_sort(int[] array) {
    int length = array.length;
    for (int i = 1; i < length; i++) {
        // 使用一个额外空间来存放一个临时变量
        int temp = array[i];
        int j = i - 1;
        // 进行连续比较，直到数组开头
        while (j >= 0 && temp <= array[j]) {
            array[j + 1] = array[j];
            j--;
        }
        array[j] = temp;
    }
    return array;
}

复杂度分析：

  • 最好情况：正序情况，正好每1个内层循环都只比较了一次，一共比较了(n-1)次，没有交换
  • 最差情况：逆序情况，正好每个内层循环比较了等差数列次，第一次比较了1次，交换了1次，第二次比
    较了2次，交换了2次···第n-2次循环比较了(n-1)次，交换了(n-1)次；第(n-1)次循环比较了(n-1)次， 交换了(n-1)次。共比较 1 + 2 + ··· + (n-2) + (n-1) = n * (n-1) / 2次，交换n(n-1)/2次
  • 平均情况：O(n^2)

稳定性：连续比较，相同不交换，稳定

结论：对于基本有序的序列，插入排序的复杂度较低，但是插入排序的效率较低，每次只移动了一位

###2.1.2 希尔排序
定义：是直接插入的改进，分组插入排序。希尔排序利用了插入排序对有序数组排序性能大增的特点，实现了计算

思路：通过选择特定的步长dl，将比较的元素分成几个区域来（进行插入排序）提升插入排序的性能。每次将区域缩小一点，最后缩小到步长dl=1的时候，就是普通的插入排序了。

代码实现：

  • 采用分割记录法，将数组分成(n/dl)个子序列
  • 对子序列进行直接插入排序
  • 步长的选择是排序关键。只要最终步长为1的所有步长序列都可以最终完成排序。算法发明者Shell推荐的步长序列是n/2，对步长取半直到步长为1

希尔排序最重要特性：对数组进行较小步长排序之后，之前用的较大步长仍然是有序的。

代码：

/**
 * 希尔排序。步长为n/2 
 * @param array 待排序数组
 * @return 排序完成数组
 */
public static int[] shell_sort(int[] array) {
    int len = array.length;
    int i;
    int j;
    int gap = len / 2;

    // 选择每一次的步长（外一层循环）；按照步长分出子序列；
    // 对每一组进行直接插入排序（内两层循环）
    for (;gap > 0; gap = gap / 2) {
        for (i = gap; i < len; i++) {
            int temp = array[i];
            for (j = i - gap; j >= 0 && array[j] > temp; j -= gap) {
                array[j + gap] = array[j];
            }
            array[j + gap]  = temp;
        }
    }
    return array;
}

复杂度分析：
步长不同，所得的时间复杂度也不同：

步长序列 最坏情况下复杂度
n/2^i O(n^2)
2^k - 1 O(n^{3/2})
2^i*3^j O( n*log^2 n)

时间复杂度：
这里采用的dlta(i) = n/2^i，时间复杂度是O(n^2)，若采用其他步长序列，则会更加高效。
平均时间复杂度为O(n^2)。
dlta(i) = 2^k-1，平均价时间复杂度为O(n^{3/2})，低于O(n^2)，这就是Shell排序划时代的地方了。

稳定性：跨越比较，相等元素之间可能交换，不稳定。

总结：希尔排序是最早打破人们思维定式，终结O(n^2)时间复杂度，突破慢速排序时间的排序算法。

##2.2 交换排序

###2.2.1 冒泡排序及其优化
定义： 对普通交换排序的改进，对无序区的记录依次进行比较，使得最大（小）的记录如同“气泡”一样“冒出水面”。

思想：通过对无序区记录进行两两比较，若反序就进行交换，一直交换到序列尾部，使元素成为有序区记录。依次交换，直到没有反序的记录为止。

代码实现：通过两层循环，从序列头部开始进行比较，反序就交换，从下往上交换。注意：尾部指的是无序区的尾部。

代码：

/**
 * 冒泡排序
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] bubble_sort(int[] array) {
    int length = array.length;
    for (int i = 0; i < length - 1; i++) {
        for (int j = 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
            }
        }
    }
    return array;
}

private static void swap(int[] array, int i, int j) {
    int temp = array[i];
    array[i] = array[j];
    array[j] = temp;
}

复杂度分析：这里要对它进行改进，之后再对其时间复杂度进行分析。

###2.2.2 优化的冒泡排序

思想：普通冒泡可能会产生许多无用的比较，尤其是在顺序情况下，强行比较；为了避免这种情况，可以记录下上一轮的比较是否进行了交换，有交换继续排序，没有则是已经有序，结束排序。

代码实现：设置一个标记位FLAG，每次外层循环设为FALSE，在内层循环中，若有一次交换就设置为TRUE。

代码：

/**
 * 改进的冒泡排序。可以避免冗余的比较和交换
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] modified_bubble_sort(int[] array) {
    int length = array.length;
    boolean flag = true;

    for (int i = 0; i < length - 1 && flag; i++) {
        flag = false;
        for (int j = 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
                flag = true;
            }
        }
    }
    return array;
}

复杂度分析：

  • 时间复杂度：
    最好情况：正序情况下，比较(n-1)次，交换0次，时间复杂度O(n)
    最坏情况：反序情况下，比较n*(n-1)/2次，交换同样多的次数，时间复杂度为O(n^2)
    平均情况：时间复杂度O(n^2)

  • 空间复杂度：原地排序，辅助空间为O(1)，交换的时候用到了辅助空间

###2.2.3 快速排序（重要）

定义：对冒泡排序的改进。

思想：通过一趟排序实现对记录的分割，其中一部分记录的关键字均比另一部分记录的关键字小。则可分别对两部分记录进行排序，已达到整体有序的。

代码实现：

  • 找到枢纽位置pivot是关键步骤，取低位元素作为枢纽值，进行循环分组。
  • 每一趟分组，先从高位进行分组，选出比枢纽值小的值，与枢纽原位置进行交换（枢纽值处于高位）；接着低位开始进行分组，选出比枢纽值大的元素，与枢纽值进行交换（枢纽值处于低位），一趟结束。
  • 若低位仍然小于高位，就继续循环，直到低位与高位相等，循环结束

代码：
/**

 * 快速排序。分治法的应用，先将数组根据第一个元素大小分成两边。
 * 一边小，一边大。再对将两边的数组分别进行同样的划分。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 排好序数组
 */
public static int[] quick_sort(int[] array, int low, int high) {
    int pivot;

    if (low < high) {
        pivot = getPartition(array, low, high);
        quick_sort(array, low, pivot - 1);
        quick_sort(array, pivot + 1, high);
    }

    return array;
}

/**
 * 分组。根据第一个元素大小分成两边的数组，一边小，一边大。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 划分好的数组
 */
private static int getPartition(int[] array, int low, int high) {
    int pivot_key = array[low];

    // 从表的两端交替向中间扫描
    while (low < high) {
        while (low < high && array[high] >= pivot_key) {
            high--;
        }

        swap(array, low, high);

        while (low < high && array[low] <= pivot_key) {
            low++;
        }

        swap(array, low, high);
    }

    return low;
}

private static void swap(int[] array, int i, int j) {
    if (i != j) {
        int temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }
}

复杂度分析：
最优情况：
时间复杂度：最优情况下，每次都很均匀地分成两组，所以最后分出来的递归树的深度就是：[log2n]+1（[x]表示不大于x的最大整数），仅需要递归.log2n.次，每次比较的次数为n次，交换的次数是n-1次，每次的时间复杂度为O(n)，每次除了比较，交换之外，还需要对其他两个子树进行排序，根据推导：
T（n）≤2T（n/2） +n，T（1）=0
T（n）≤2（2T（n/4）+n/2） +n=4T（n/4）+2n
T（n）≤4（2T（n/8）+n/4） +2n=8T（n/8）+3n
……
T（n）≤nT（1）+（log2n）×n= O(nlogn)
所以时间复杂度为O(nlog2n).

空间复杂度：
每次递归都需要1个额外空间，所以空间复杂度为O(log2n)，

最坏情况：
时间复杂度：最坏情况下，序列是反序的，每次分组都是比上一次少划分一个记录的子序列，另一边则为空，这样就需要进行n-1次分组，递归树深度为n-1，每次比较了n-i次，一共比较了n*(n-1)/2次，所以时间复杂度为O(n^2)

###2.2.4 改进的快速排序
快速排序存在两个问题：
1) 枢轴
问：为什么在正序或者反序的时候，快速排序的时间复杂度为平方阶O(n^2)？
答： 因为对于这两种情况，一边的数组是空的，另一边又是待排序数组的完整版。而且分完组的话枢纽值要么最终在头上，要么最终在尾巴上，比较次数和比较的循环轮数都是最大的。最后的时间复杂度达到了O(n^2)。

2) 递归交换
问：这里面的分组函数getPartition每次调用swap都会进行3次操作，能不能改成替换，只进行一次操作？
答：可以。

3) 性能问题（小数据样本时）
问：当数组数量低于7个（资料显示）的时候，采用插入排序反而效率较高？
答：博士生去小学教“1+1=2”也不见得比本科生教得好。快速排序对于基本有序，或者样本极少的数组排序的确效率不高。插入排序反而比较适合基本有序的小样本排序。

4) 递归交换
问：大家知道，递归对性能是有一定影响的，quick_sort函数在其尾部有两次递归操作。如果待排序的序列划分极端不平衡（正序或反序），递归深度将趋近于n，而不是平衡时的log2n，栈的空间是极其有限的，如何减少递归？
答：解决递归地最好方法就是使用迭代。所以我们需要优化这里面不必要的递归交换。

1)问题1解决方案：
修改getPartition函数。枢轴不合理，那就选取合理的枢轴。现有的方案中：随机选取比较耗费性能，而且还不能保证一定会简化排序；九数取中分三路进行三路取中，较为复杂，先不讨论；这里采用三数取中选取枢轴值。

/**
 * 分组。先进行三数取中选取枢轴值，再根据枢轴值分成两个数组，一边小，一边大。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 划分好的数组
 */
private static int getPartition1(int[] array, int low, int high) {
    int pivot_key;
    // 进行三数取中
    int mid = low + (high - low) / 2;
    // 先确保左边是比右边小
    if (array[low] > array[high]) {
        swap(array, low, high);
    }
    // 再确保中间比右边小
    if (array[mid] > array[high]) {
        swap(array, mid, high);
    }
    // 最后确保左边比中间小
    if (array[low] > array[mid]) {
        swap(array, low, mid);
    }
    pivot_key = array[low];

    // 从表的两端交替向中间扫描
    while (low < high) {
        while (low < high && array[high] >= pivot_key) {
            high--;
        }

        swap(array, low, high);

        while (low < high && array[low] <= pivot_key) {
            low++;
        }

        swap(array, low, high);
    }

    return low;
}

修改quick_sort函数。那就加入判断，小于等于7个的时候调用插入排序，大于7个采用快速排序。

private static final int MAX_LENGTH_INSERT_SORT = 7;
/**
 * 改进的快速排序。在数据较少时采用递归。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 排好序数组
 */
public static int[] modified_quick_sort1(int[] array, int low, int high) {
    int pivot;

    if (array.length > MAX_LENGTH_INSERT_SORT) {
        if (low < high) {
            pivot = getPartition(array, low, high);
            quick_sort(array, low, pivot - 1);
            quick_sort(array, pivot + 1, high);
        }
    } else {
        // 调用直接插入排序
        DirectlyInsertSort.directly_insert_sort(array);

    return array;
}

4)问题4解决方案：
修改quick_sort函数。将if改成while，使用迭代，删除第二次递归，然后将之前荒废掉的low利用起来，作为指针使用。

/**
 * 改进的快速排序。优化了递归。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 排好序数组
 */
public static int[] modified_quick_sort2(int[] array, int low, int high) {
    int pivot;

    // 将递归换成迭代，减少对栈的使用
    while (low < high) {
        pivot = getPartition(array, low, high);
        quick_sort(array, low, pivot - 1);

#2.3 选择排序

##2.3.1 简单选择排序
思想：每一趟从未排序元素中选择出一个最小（最大）的值，放在队头。

代码实现：使用两层循环，每次用min（max）保存一个最值，对未排序元素进行依次比较，得出的结果与未排序元素头元素进行交换，然后又指针（数组下标）往后移动一位，形成新的无序元素表，循环直到剩余元素为空。

代码：

public static int[] simple_select_sort(int[] array) {
    int length = array.length;
    int pos;
    for (int i = 0; i < length - 1; i++) {
        pos = i;
        for (int j = i + 1; j <= length - 1;j++) {
            if (array[j] < array[pos]) {
                pos = j;
            }
        }
        // 把最小元素换到队头来
        swap(array, i, pos);
    }
    return array;
}

复杂度分析：
时间复杂度：最好，最坏比较都是(N-1)*(N+1)/2,交换是N-1次，所以时间复杂度是O(n^2)
空间复杂度：原地排序，一个临时变量，所以是常数级别的O(1)
稳定性:因为是相邻元素的比较，而且中途没有交换，所以是稳定的，而且是特别稳定，任何情况的比较和交换（不算赋值过程）都是一样的

设数组为a[0…N-1]
根据完全二叉树的性质：非叶子节点i的左右叶子节点（存在）下标分别为2i+1,2i+2。数组的最后一个非叶子节点为N/2-1。
实现堆排序步骤如下：
1.得到大根堆或者小根堆。这里以大根堆为例：
我们可以把每个非叶子节点都看成一个完全二叉树，然后都生成一个大根堆，得到一个初始大根堆堆。
2.取出生成大根堆的顶点值，与堆中第N个元素，也就是a[N-1]交换
3.此时大根堆被破坏，再重新生成大根堆，容量N–
4.重复2,3，直到N=1，排序完成

/**
  * 堆排序
  * @param array 待排序数组
  * @return
  */
 public static int[] heap_sort(int[] array) {
     int N = array.length;
     if (N < 2) {
         return array;
     }
     for (int i = N/2 - 1; i >= 0; i--) {
         fitToMaxHeap(array, i, N - 1);
     }
     for (int i = N - 1; i > 0; i--) {
         swap(array, 0, i);
         fitToMaxHeap(array, 0, i - 1);
     }
     return array;
 }
 /**
  * 调整成为大根堆
  * @param array
  * @param top 堆顶元素
  * @param tail 堆尾部元素
  */
 private static void fitToMaxHeap(int[] array, int top, int tail) {
     int max = array[top];
     // 沿着关键字较大的孩子节点向下
     for (int i = 2 * top + 1; i <= tail; i = 2 * i + 1) {
         if (i < tail && array[i + 1] > array[i]) {
             ++i;
         }
         // 如果这是最大的，直接跳出，如果不是，交换元素
         if (max >= array[i]) {
             break;
         }
         // 顶点也要移动
         array[top] = array[i];
         top = i;
     }
     array[top] = max;
 }
 private static void swap(int[] array, int i, int j) {
     if (i != j) {
         int temp = array[i];
         array[i] = array[j];
         array[j] = temp;
     }
 }

复杂度分析：
运行时间主要在排序阶段。先是建立一个初始堆，需要时间O(n)（N/2-1）时间。然后第i次建立堆得比较和交换次数都是log2(i)次，而一共排序(n-1)次，所以排序的时间复杂度为O(nlogn)。
空间复杂度：O(1)，原地排序，辅助空间就一个temp

稳定性：
因为是父节点跟子节点的比较，不会连续，所以堆排序是不稳定的。

#2.5 归并排序

##2.5.1 原始的归并排序
预备知识：

  1. 归并在数据结构中的定义是：将两个或者两个以上的有序表组合成一个新的有序表。
  2. 归并算法不唯一：多路归并算法有多种实现它的路数是浮动的K，我们在这里讲的是最简单的二路归并。
  3. 分治法：将一个复杂的问题不断地分成两个或者更多相同或相似的子问题来解决，直到最后子问题可以直接简单求解为止，子问题的解得合并即为原问题的解。

思想：二路归并算法是利用分治法的思想。它假设初始序列有N个记录，将这N个元素看做N个有序子序列，每个子序列的长度为1，然后进行合并。

/**
 * 归并排序
 * @param array 待排序序列
 * @return 已排序序列
 */
public static int[] merge_sort(int[] array) {
    int length = array.length;
    if (length <= 1) {
        return array;
    }
    length = array.length;
    int[] target = new int[length];

    return m_sort(array, target, 0, length - 1, length);
}

/**
 * 归并排序算法
 * @param array 待排序序列
 * @param target 排好序输出的目标序列
 * @param start 排序起始位置
 * @param end 排序末尾位置
 * @param length 待排序序列长度
 * @return 排好序序列
 */
private static int[] m_sort(int[] array, int[] target, int start, int end, int length) {
    int middle;
    int[] temp = new int[length];

    if (start == end) {
        target[start] = array[start];
    } else {
        middle = (start + end) / 2;

        m_sort(array, temp, start, middle, length);
        m_sort(array, temp, middle + 1, end, length);
        // 两两归并
        merge(temp, target, start, middle, end);
    }

    return target;
}

/**
 * 将数组temp[start...middle]和temp[middle+1...end]归并起来
 * 其实就是讲两个序列有序地合并，先在两个序列都不为空的情况下进行元素比较，
 * 哪个小插入哪个到尾部，直到其中至少一个序列为空，然后将剩余的序列依次插入尾部
 * 这里面的两个序列就来自于数组temp的前半（start...middle）和后半（middle+1...end）
 *
 * @param temp 待排序总序列
 * @param target 排好序之后输出的序列
 * @param start 前半序列起始位置
 * @param middle 中间分隔两个序列的位置，前半序列的中点位置，后半序列起始位置的前一个位置
 * @param end 后半序列的末尾位置
 */
private static void merge(int[] temp, int[] target, int start, int middle, int end) {
    int i = start;
    int j = middle + 1;
    int k = start;// target指针

    // 有序地合并序列，特别注意，这里target的插入起始位置跟temp的起始位置是一样的
    while (i <= middle && j <= end) {
        if (temp[i] < temp[j]) {
            target[k++] = temp[i++];
        } else {
            target[k++] = temp[j++];
        }
    }

    // 若前半没合并完，依次插入尾部
    while (i <= middle) {
        target[k++] = temp[i++];
    }

    // 若后半没合并完，依次插入尾部
    while (j <= end) {
        target[k++] = temp[j++];
    }
}

复杂度：
时间复杂度：首先， 归并排序需要将temp[0]…temp[n]中相邻长度为h的所有记录两两排序归，并存放到target[0]…target[n]里面去。此过程，需要将temp所有记录都扫描一遍，因此耗费O(n)的时间，而由完全二叉树的深度为[log2(n)]+1可知，归并排序需要进行[log2(n)]的常数倍次，因此，总的时间复杂度为O(nlog2(n))，而且是最好、平均、最坏都是如此。

空间复杂度：存放扫描后数据的空间为O(n)，扫描过程中递归需要的栈空间为O(logn)，所以时间复杂度为O(n+logn)。排序空间耗费很大，这是归并排序的一大缺陷。

稳定性：因为归并排序是相邻数字两两交换，所以说是稳定的。

##2.5.2 改进的归并排序

问题的提出：因为空间耗费大是归并排序的一大缺陷，而且这一缺陷是递归造成的。所以在这里，可以将递归优化成迭代。减少空间占用。

思想：将递归换成迭代，利用新创建的target数组与源数组辗转排序最终完成排序。

代码：

 /**
 * 归并排序算法。从最小的间距为2开始，对数组进行迭代排序
 * @param array 待排序序列
 * @return 排好序序列
 */
private static int[] modified_m_sort(int[] array) {
    int interval = 1;
    int length = array.length;
    int[] target = new int[length];

    // 之所以调用两次。1.非原地排序，需把两个序列来回倒换
    // 2.即使interval长度超过了序列总长度，也有判断可以避免错误indexOutOfRange
    while (interval < length - 1) {
        mergePass(array, target, interval, length);
        interval *= 2;

        if (interval > length - 1) {
            return target;
        } else {
            mergePass(target, array, interval, length);
            interval *= 2;
        }
    }

    return array;
}

/**
 * 通过对序列里面的子序列进行排序进行，从头到尾，将标准长度的子序列两两归并
 * @param srcArray 待排序序列
 * @param targetArray 排好序序列
 * @param interval 每个归并子序列标准长度
 * @param length 待排序序列总长度
 */
private static void mergePass(int[] srcArray, int[] targetArray, int interval, int length) {
    int i = 0;
    int j;
    // 以两个间隔为单位，进行归并排序
    while (i <= length - 1 - (2 * interval - 1)) {
        merge(srcArray, targetArray, i, i + interval - 1, i + 2 * interval - 1);

        i += 2 * interval;
    }

    // 还剩不止一个子序列，进行两两归并
    if (length - 1 - i > interval - 1) {
        merge(srcArray, targetArray, i, i + interval - 1, length - 1);
    } else {
        // 最后若只剩下单个子序列，直接插入尾
        for (j = i; j < length; j++) {
            targetArray[j] = srcArray[j];
        }
    }
}

/**
 * 将数组temp[start...middle]和temp[middle+1...end]归并起来
 * 其实就是讲两个序列有序地合并，先在两个序列都不为空的情况下进行元素比较，
 * 哪个小插入哪个到尾部，直到其中至少一个序列为空，然后将剩余的序列依次插入尾部
 * 这里面的两个序列就来自于数组temp的前半（start...middle）和后半（middle+1...end）
 *
 * @param temp 待排序总序列
 * @param target 排好序之后输出的序列
 * @param start 前半序列起始位置
 * @param middle 中间分隔两个序列的位置，前半序列的中点位置，后半序列起始位置的前一个位置
 * @param end 后半序列的末尾位置
 */
private static void merge(int[] temp, int[] target, int start, int middle, int end) {
    int i = start;
    int j = middle + 1;
    int k = start;// target指针

    // 有序地合并序列，特别注意，这里target的插入起始位置跟temp的起始位置是一样的
    while (i <= middle && j <= end) {
        if (temp[i] < temp[j]) {
            target[k++] = temp[i++];
        } else {
            target[k++] = temp[j++];
        }
    }

    // 若前半没合并完，依次插入尾部
    while (i <= middle) {
        target[k++] = temp[i++];
    }

    // 若后半没合并完，依次插入尾部
    while (j <= end) {
        target[k++] = temp[j++];
    }
}

复杂度：
因为没有使用递归，所以这里的空间复杂度降低了，但是在interval最大的一次归并还是需要扫描整个序列的一半以上，所以空间复杂度仍然是O(n)，但是相比于递归已经好很多了。
时间复杂度：因为i是以2为底的指数，所以时间复杂度和改进之前的一样，都是O(nlogn)。所以，我们一半使用的归并排序都是非递归形式的归并排序。

排序算法优化的原则：
1.能用迭代就不要用递归。
2.能用替换就不要用交换(swap)
3.低级排序并不一定比高级排序差，主要还是要看清楚应用场景

#3 外排序

  • 定义：外排序（External sorting）是指能够处理极大量数据的排序算法。

通常来说，外排序处理的数据不能一次装入内存，只能放在读写较慢的外存储器（通常是硬盘）上。外排序通常采用的是一种“排序-归并”的策略。在排序阶段，先读入能放在内存中的数据量，将其排序输出到一个临时文件，依此进行，将待排序数据组织为多个有序的临时文件。尔后在归并阶段将这些临时文件组合为一个大的有序文件，也即排序结果。

堆排序

因为堆排序的时空复杂度是比较特殊和有趣的，所以这里单拎出来说一说。

  • 堆排序的空间复杂度O(1)，因为只使用了一个辅助空间。

  • 堆排序的时间复杂度，建堆时间O(N)，调整堆时间O(Nlog2(N))。对于堆排序的时间复杂度。可以分为两个部分分别讨论。

  1. 调堆时间，因为一共N个元素所以树的高度H=log2(N)+1（性质4），每次是从根节点开始比较，比较最多比较了2(H-1)次，交换的最坏情况调整了(H-1)，那么一共有N-1个元素需要调整，可以估算时间复杂度就是O(Nlog2(N))。
  2. 关键是建堆时间，因为是从最后一个非叶子结点开始调整，所以肯定调整了与N同量级的次数，时间复杂度不可能是log2(N)(H)。但是会不会是O(Nlog2(N))？因为每次最后一个最后一个非叶子结点入堆顶都得使用时间为(log2(N)-1)的时间，而非叶子结点的个数为N/2，那么O(Nlog2(N))看起来很合理不是吗？
    实际上不是的，因为不是从根开始比较而是从第一个非叶子结点开始比较，比较次数少了很多。具体计算，我们从底层开始算起，最后一层（H层）父节点一共有2^(H-1)个结点，他们最多需要交换的次数是1次，比较的次数是2次；倒数第二层的一共有2^(H-2)个结点，他们最多需要交换的次数是2次，比较的次数是4次，倒数第二层父节点的个数是2个，他们最多需要交换的次数是H-2次，比较的次数是2(H-2)次；根节点就1个，它最多需要交换的次数是(H-1)次，比较的次数是2(H-1)次。

所以说，最坏情况下的交换次数是S = 1 2^(H-1) + 2 2^(H-2) + ··· + (H-2) 2^1 + (H-1) 2^0 = 2N - 2 - log2(N),最坏情况下比较的次数是 2S 次，所以时间复杂度是O(3S = 6N - 6 - 3log2(N))，即为O(N);
没有排序是完美的，堆排序尽管非常优秀，有很高效的时间复杂度（平均O(nlogn)），而且有很低的空间复杂度O(1)，但是由于非连续比较的缘故，并不是稳定的排序算法。

附加专题：完全二叉树的5个性质推导

  1. 所有二叉树的第i层上最多拥有2^(i-1)个节点。
    证明：数学归纳法
    1层 1个 2^0
    2层 2个 2^1
    …
    假设 k层 2^k个
    k+1 层 2*2^(k+1) = 2^(k+1)个【父节点下最多拥有两个子节点】
    …
    n层 2^n个
    所以，性质成立。
  2. 二叉树中如果深度为k,那么最多有2^k - 1个节点
    证明：根据性质1，求k层满二叉树的节点个数，个数s = (1+2+…+2^(k-1)) = 1 * (1 - 2^k)/ (1 - 2) = 2^k - 1。
    注释：满二叉树就是每一层都是满节点的特殊二叉树，是完全二叉树的究极形态。
  3. n0=n2+1 n0表示度数为0的节点 n2表示度数为2的节点
    证明：根据两个公式：
    n = n0 + n1 + n2
    n - 1 = n1 + 2 n2 入度等于出度，入度是除了根节点外其他节点都为1，出度是按定义计算。
    两个公式通过n作为桥梁，就可以化简为 (n0 + n1 + n2) - 1 = n1 + 2     n2
    得到n0=n2+1
    1. 在完全二叉树中，具有n个节点的完全二叉树的高度是[log2(n)] + 1，其中，[log2(n)] 是向下取整。
      证明：我们假设这个完全二叉树的高度为k.
      对一个高度为k的完全二叉树而言，它的节点数一定比高度为(k-1)的满二叉树要多（大于），小于等于高度为k的满二叉树。根据性质2，可以得不等式 2^(k-1) - 1 < n <= 2^k - 1。因为n为整数，
      所以不等式可以大胆地加减1，得：2^(k-1) -1 + 1 <= n < 2^k - 1 + 1 。
      对不等式去以二为底的对数，得： k - 1 <= log2(n) < k。所以 k = [log2(n)] + 1，其中，[log2(n)] 是向下取整。
    2. 如果有一颗有n个节点的完全二叉树的节点按层次序编号，对任一层的节点i（1<=i<=n）有：
    • 如果i = 1，则节点为根节点，无双亲，如果i > 1， 那么它的双亲为[i/2]，向下取整
    • 如果2i > n，节点i没有左孩子节点，否则左孩子节点为2i
    • 如果2i + 1 > n，节点i没有右孩子节点，否则右孩子节点为2i + 1
    • 最后一个非叶子节点编号为 n/2

第五条：对Java而言，数组从0开始，那么规律就需要变化一下

  • 如果i = 0，则节点为根节点，无双亲，如果i > 0， 那么它的双亲为[(i - 1)/2]，向下取整
  • 如果2i + 1 > n，节点i没有左孩子节点，否则左孩子节点为2i + 1
  • 如果2i + 2 > n，节点i没有右孩子节点，否则右孩子节点为2i + 2
  • 最后一个非叶子节点编号为 n/2 - 1

    现在仅仅完成了第一阶段，第二阶段完成以后再来对代码进行解析。第一阶段的设计思路如下:

/**

  • IP分配算法，现在一个三层网络的地址池，和为二层准备的子网前缀，还有一些可能已经分配出去的子网地址
  • 为当前的子网分配一个合适的子网
  • 第一阶段，不支持修改。支持修改的话存放的所有的字符串就都是子网，而不是ip了
  • 第二阶段，支持修改。
  • 解决思路：写一个合并和拆分子网的算法，然后每次从自由地址池里面分配之后，需要对自由地址进行子网的合并和拆分
  • 每次正常添加子网的时候也需要计算后面的掩码
    *
  • @Author Dream Wu
  • @Date 2017/5/27
  • @Email wzl961@163.com
    */

Java代码如下：

public class SubnetAllocated {
    // 当前最新子网cidr的ip部分
    public static String currentGreatestIp;
    // 存放所有已分配子网
    public static Map<String, List<String>> subnetIpMap = new HashMap<>();
    public static Map<String, Boolean> ipPoolUsableMap = new HashMap<>();
    //自由子网地址池，只有在部分子网创建后被释放的情况下会被使用
    public static Map<String, List<String>> freeSubnetIpMap = new HashMap<>();

public static final Integer IP_SEGMENT_LENGTH = 8;

public static void main(String args[]) {
    long startTime=System.currentTimeMillis();
    System.out.println(ipToSubnet("10000000100000001000000010000000", 24));
    long endTime=System.currentTimeMillis();
    System.out.println("Run Time :" + (endTime-startTime));
}

public static String allcateSubnet(String ipPool, Integer subnetMask) {
    // 将地址池分为ip和掩码，分别进行处理
    String[] subnet = ipPool.split("\\/");
    // 地址池的上限，如果达到次上限，子网就算分完了
    String ipPoolLimit;
    // 分配好的子网的IP
    String allocatedIp;
    // 分配好的子网
    String allocatedSubnet = null;
    // ip地址池的二进制字符串
    String binaryIpPool;
    // 地址池的掩码部分
    Integer ipPoolMask = 0;
    // 分配成功的标志
    boolean flag = false;

    // 第一步，判断subnetMask是否合法
    if (subnet[1] != null) {
        try {
            ipPoolMask = Integer.valueOf(subnet[1]);
        } catch (Exception e) {
            System.out.println("mask Exception : " + e.getMessage());
            return null;
        }
    }

    if (ipPoolMask >= subnetMask) {
        System.out.println("subnetMask should be less than ip pool mask.");
        return null;
    }

    // 第二步，将地址池的ip部分转化成二进制数，并且将其与地址池的上限进行比较
    try {
         binaryIpPool = ipToBinary(subnet[0]);
    } catch (Exception e) {
        System.out.println("Ip string illegal.Exception : " + e.getMessage());
        return null;
    }

    // 第三步，得到地址池的上限，并且对第一次分配子网进行初始化
    ipPoolLimit = binaryAdd(binaryIpPool, ipPoolMask);

    if (currentGreatestIp == null) {
        currentGreatestIp = binaryIpPool;
        allocatedIp = currentGreatestIp;
    } else {
        allocatedIp = binaryAdd(currentGreatestIp, subnetMask);
    }

    // 第四步，IP的最终分配。满足条件：
    // 1. 分配的IP小于上限IP
    // 2. 分配的IP未被覆盖到
    // 3. 还有剩余子网

    // 第一次给地址池分IP，先分了再说
    if (subnetIpMap.get(ipPool) == null) {
        List<String> ipList = new LinkedList<>();
        ipList.add(allocatedIp);
        subnetIpMap.put(ipPool, ipList);
    }
    if (ipPoolUsableMap.get(ipPool) == null) {
        ipPoolUsableMap.put(ipPool, true);
        return ipToSubnet(allocatedIp, subnetMask);
    }

    //地址池已经分完，不可再分
    if (!ipPoolUsableMap.get(ipPool)) {
        System.out.println("No subnet available.");
        return null;
    }

    // 第二次及以后分配ip, 没成功就一直分
    // 若在范围内就一直加
    if (!checkSubnetOverIpRange(allocatedIp, ipPoolLimit)) {
        while (!flag) {
            if (!checkSubnetIpIsInUsed(allocatedIp, ipPool)) {
                List<String> ipList = subnetIpMap.get(ipPool);
                ipList.add(allocatedIp);
                subnetIpMap.put(ipPool, ipList);
                currentGreatestIp = allocatedIp;
                allocatedSubnet = ipToSubnet(allocatedIp, subnetMask);

                flag = true;
            } else {
                allocatedIp = binaryAdd(allocatedIp, subnetMask);
            }
        }
    } else {
        // 若allocatedIp超过上限，就从自由的地址池里面分
        if (freeSubnetIpMap.get(ipPool) != null && freeSubnetIpMap.get(ipPool).size() != 0) {
            allocatedIp = binaryIpPool;

            List<String> freeIpList = freeSubnetIpMap.get(ipPool);

            while (!flag) {
                if (freeIpList.contains(allocatedIp)) {
                    freeIpList.remove(allocatedIp);
                    freeSubnetIpMap.put(ipPool, freeIpList);

                    List<String> ipList =  subnetIpMap.get(ipPool);
                    ipList.add(allocatedIp);
                    subnetIpMap.put(ipPool, ipList);
                    currentGreatestIp = allocatedIp;
                    allocatedSubnet = ipToSubnet(allocatedIp, subnetMask);

                    flag = true;
                } else {
                    allocatedIp = binaryAdd(allocatedIp, subnetMask);
                }
            }
        } else {
            // 自由子网也分配完了就是没有真的地址了
            ipPoolUsableMap.put(ipPool, false);
            System.out.println("No subnet available.");
            return null;
        }
    }

    return allocatedSubnet;
}

// 判断allocatedIp是否被使用或者超过了范围
private static boolean checkSubnetIpIsInUsed(String allocatedIp, String ipPool) {
    List<String> ipList = subnetIpMap.get(ipPool);
    return ipList.contains(allocatedIp);
}

// 判断allocatedIp是否超越了上限
private static boolean checkSubnetOverIpRange(String allocatedIp, String ipPoolLimit) {
    // 获取allocatedIp的二进制字符串字段
    int[] allocateNum = new int[4];
    int[] limitNum = new int[4];

    for (int i = 0; i < allocateNum.length; i++) {
        try {
            allocateNum[i] = Integer.valueOf(allocatedIp.substring(i * 8, (i+1) * 8), 2);
            limitNum[i] = Integer.valueOf(ipPoolLimit.substring(i * 8, (i+1) * 8), 2);
        } catch (Exception e) {
            System.out.println("Ip string illegal.Exception : " + e.getMessage());
            return true;
        }
    }

    // 比较两个子网ip的大小,一旦allocatedIp大于等于ipPoolLimit，返回false
    if (allocateNum[0] > limitNum[0]) {
        return true;
    } else if (allocateNum[0] == limitNum[0]) {
        if (allocateNum[1] > limitNum[1]) {
            return true;
        } else if (allocateNum[1] == limitNum[1]) {
            if (allocateNum[2] > limitNum[2]) {
                return true;
            } else if (allocateNum[2] == limitNum[2]) {
                if (allocateNum[3] > limitNum[3]) {
                    return true;
                } else if (allocateNum[3] == limitNum[3]) {
                    return true;
                }
            }
        }
    }

    return false;
}


// 将二进制的ip地址和子网掩码的二进制数相加，得到的就是一个子网的ip
private static String binaryAdd(String binaryIp, Integer mask) {
    //先将掩码转换成二进制数
    Integer middle = 1;
    String binaryMask;
    String allocatedIp;

    for (int i = 0; i < 32 - mask; i++) {
        middle *= 2;
    }
    binaryMask = Integer.toBinaryString(middle);

    int[] ca = new int[binaryIp.length()];
    int[] cb = new int[binaryMask.length()];

    // 将字符数组转换成数值的0或者1
    for (int i = 0; i < binaryIp.length(); i++) {
        ca[i] = binaryIp.charAt(i) - '0';
    }
    for (int i = 0; i < binaryMask.length(); i++) {
        ca[i] = binaryMask.charAt(i) - '0';
    }

    // 用ca保存长度的长
    if (ca.length < cb.length) {
        int[] temp = ca;
        ca = cb;
        cb = temp;
    }

    // 字符数组中最后一个索引下标，也是最小的一个
    int la = ca.length - 1;
    int lb = cb.length - 1;

    // 下位的进位标识符
    int carry = 0;

    //两位相加的结果
    int result;

    while (la >= 0 && lb >= 0) {
        result = ca[la] + cb[lb] + carry;

        ca[la] = result % 2;
        carry = result / 2;

        la--;
        lb--;
    }

    // 处理余下的数字
    while (la >= 0) {
        result = ca[la] + carry;
        ca[la] = result % 2;
        carry = result / 2;

        if (la == 0) {
            break;
        }

        la--;
    }

    // 将计算结果换回字符
    char[] chars = new char[ca.length];
    for (int i = 0; i < ca.length; i++) {
        ca[i] += '0';
        chars[i] = (char)ca[i];
    }

    return String.valueOf(chars);
}

// 将十进制的字符串ip转化成二进制数
private static String ipToBinary(String ip) {
    String[] binaryIps = new String[4];

    // 先将ip切分成4个ip段
    if (ip != null) {
        String[] ips = ip.split("\\.");

        for (int i = 0; i < ips.length; i++) {
            try {
                binaryIps[i] = Integer.toBinaryString(Integer.valueOf(ips[i]));
            } catch (Exception e) {
                throw new IllegalArgumentException();
            }
        }
    } else {
        System.out.println("Ip string illegal!");
        return null;
    }

    // 将4个ip段分别转化成2进制再进行拼接，得到一个32位的数字
    StringBuilder stringBuilder = new StringBuilder(1024);
    for (String binaryIp : binaryIps) {
        String ipSegment;
        char[] ipChars = binaryIp.toCharArray();
        char[] binaryChars = new char[IP_SEGMENT_LENGTH];

        for (int j = 0; j < binaryChars.length; j++) {
            if (j < ipChars.length) {
                binaryChars[binaryChars.length - 1 -j] = ipChars[j];
            } else {
                binaryChars[binaryChars.length - 1 -j] = '0';
            }
        }
        ipSegment = String.valueOf(binaryChars);
        stringBuilder.append(ipSegment);
    }

    return stringBuilder.toString();
}

// 二进制IP转化成标准子网形式
private static String ipToSubnet(String allocatedIp, Integer subnetMask) {
    // 获取allocatedIp的二进制字符串字段
    Integer[] allocateNum = new Integer[4];
    StringBuilder sb = new StringBuilder(1024);

    for (int i = 0; i < allocateNum.length; i++) {
        try {
            allocateNum[i] = Integer.valueOf(allocatedIp.substring(i * 8, (i+1) * 8), 2);
        } catch (Exception e) {
            System.out.println("Ip string illegal.Exception : " + e.getMessage());
            return null;
        }
        if (i == allocateNum.length - 1) {
            sb.append(allocateNum[i]).append("/").append(subnetMask.toString());
            continue;
        }
        sb.append(allocateNum[i]).append(".");
    }

    return sb.toString();
    }
}
