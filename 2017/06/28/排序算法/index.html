比较排序算法

1.前言

  根据排序过程中待排序记录是否全部存放在内存中，可以将排序算法归为两类：内排序和外排序。一般的记录数较少的情况下，采用的方法都是内排序；而记录数量巨大，而且要求排序算法效率高的情况下，采用的是外排序。

常见的内外排序有哪些？
内排序：插入排序算法、选择排序算法、交换排序算法、归并排序算法
外排序：外归并排序算法

比较排序算法有哪些？性能表现又是如何？

注1：关于 Memory，如果算法为 “in place”（原地） 排序，则仅需要 O(1) 内存；有时对于额外的 O(log(n)) 内存也可以称为 “in place”（原地）。

注2：Microsoft .Net Framework中Array.Sort方法就是实现了内省排序算法。

2.内排序

2.1插入排序 （Insertion Sorts）

2.1.1 直接插入排序 (Directly Insertion Sort)

定义：将一个记录插入一个有序表中，从而得到一个新的，记录数加一的有序表。

思路：无序插入有序表，每次从位置K由后向前进行比较，找到第一个不大于（小于）A[K]的数字，否则就放在数组头部。

复杂度分析：

  • 最好情况：正序情况，正好每1个内层循环都只比较了一次，一共比较了(n-1)次，没有交换
  • 最差情况：逆序情况，正好每个内层循环比较了等差数列次，第一次比较了1次，交换了1次，第二次比
    较了2次，交换了2次···第n-2次循环比较了(n-1)次，交换了(n-1)次；第(n-1)次循环比较了(n-1)次， 交换了(n-1)次。共比较 1 + 2 + ··· + (n-2) + (n-1) = n * (n-1) / 2次，交换n(n-1)/2次
  • 平均情况：O(n^2)

稳定性：连续比较，相同不交换，稳定

评价：对于基本有序的序列，插入排序的复杂度较低，但是插入排序的效率较低，每次只移动了一位

代码实现：采用双层循环，外层循环每次将数据跟前面的有序表进行对比，如果小于，就交换数字，往前对比；若不小于，则停止前进。
注释：这里的话作为交换条件的temp和array[j]的比较加到到循环里面去合理一些，所以说内层循环使用while更加简洁。

代码：

/**
 * 直接插入排序
 * @param array 待排序数组
 * @return 排序完成数组
 */
public static int[] directly_insert_sort(int[] array) {
    int length = array.length;
    for (int i = 1; i < length; i++) {
        // 使用一个额外空间来存放一个临时变量
        int temp = array[i];
        int j = i - 1;
        // 进行连续比较，直到数组开头
        while (j >= 0 && temp < array[j]) {
            array[j + 1] = array[j];
            j--;
        }
        array[j + 1] = temp;
    }
    return array;
}

        // 进行连续比较，直到数组开头

            array[j + 1] = array[j];
            j--;
        }
        array[j + 1**粗体文本**] = temp;

注释：这里使用了while而非for，让循环看起来更加清晰。while相比于for，更加强调对循环条件的满足，而插入排序的核心思想就是：把小于等于要插入数字的所有数字往后移一位， 然后插入数字，为保证移动和判断的连贯性，使用了while循环。

2.1.2 希尔排序 (Shell Sort)

定义：是直接插入的改进，分组插入排序。希尔排序利用了插入排序对有序数组排序性能大增的特点，实现了计算

思路：通过选择特定的步长dl，将比较的元素分成几个区域来（进行插入排序）提升插入排序的性能。每次将区域缩小一点，最后缩小到步长dl=1的时候，就是普通的插入排序了。

复杂度分析：
步长不同，所得的时间复杂度也不同：

步长序列 最坏情况下复杂度
n/2^i O(n^2)
2^k - 1 O(n^{3/2})
2^i*3^j O( n*log^2 n)

时间复杂度：
这里采用的dlta(i) = n/2^i，时间复杂度是O(n^2)，若采用其他步长序列，则会更加高效。
平均时间复杂度为O(n^2)。
dlta(i) = 2^k-1，平均价时间复杂度为O(n^{3/2})，低于O(n^2)，这就是Shell排序划时代的地方了。

稳定性：跨越比较，相等元素之间可能交换，不稳定。

评价：希尔排序是最早打破人们思维定式，终结O(n^2)时间复杂度，突破慢速排序时间的排序算法。

代码实现：

  • 采用分割记录法，将数组分成(n/dl)个子序列
  • 对子序列进行直接插入排序
  • 步长的选择是排序关键。只要最终步长为1的所有步长序列都可以最终完成排序。算法发明者Shell推荐的步长序列是n/2，对步长取半直到步长为1

希尔排序最重要特性：对数组进行较小步长排序之后，之前用的较大步长仍然是有序的。

代码：

/**
 * 希尔排序。步长为n/2 
 * @param array 待排序数组
 * @return 排序完成数组
 */
public static int[] shell_sort(int[] array) {
    int len = array.length;
    int i;
    int j;
    int gap = len / 2;

    // 选择每一次的步长（外一层循环）；按照步长分出子序列；
    // 对每一组进行直接插入排序（内两层循环）
    for (;gap > 0; gap = gap / 2) {
        for (i = gap; i < len; i++) {
            int temp = array[i];
            for (j = i - gap; j >= 0 && array[j] > temp; j -= gap) {
                array[j + gap] = array[j];
            }
            array[j + gap]  = temp;
        }
    }
    return array;
}

关键代码：

                    array[j + gap] = array[j];
                }
                array[j + gap]  = temp;

注释：三层循环，最里层是一个增量为gap的插入排序，里层的是一个从步长位置开始（因为之前位置没有意义）的排序，并且每次循环结尾都会把待插入的值插入最终位置。最外层就是变步长的循环，用于不断缩小步长，逼近有序。

2.2 交换排序 (Exchange Sort)

2.2.1 普通冒泡排序 (Bubble Sort)

定义： 对普通交换排序的改进，对无序区的记录依次进行比较，使得最大（小）的记录如同“气泡”一样“冒出水面”。

思想：通过对无序区记录进行两两比较，若反序就进行交换，一直交换到序列尾部，使元素成为有序区记录。依次交换，直到没有反序的记录为止。

复杂度分析：这里要对它进行改进，之后再对其时间复杂度进行分析。

稳定性：因为是相邻交换，所以冒泡排序是稳定排序。

代码实现：通过两层循环，从序列头部开始进行比较，反序就交换，从下往上交换。注意：尾部指的是无序区的尾部。

代码：

/**
 * 冒泡排序
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] bubble_sort(int[] array) {
    int length = array.length;
    for (int i = 0; i < length - 1; i++) {
        for (int j = 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
            }
        }
    }
    return array;
}

private static void swap(int[] array, int i, int j) {
    int temp = array[i];
    array[i] = array[j];
    array[j] = temp;
}

2.2.2 优化的冒泡排序

思想：普通冒泡可能会产生许多无用的比较，尤其是在顺序情况下，强行比较；为了避免这种情况，可以记录下上一轮的比较是否进行了交换，有交换继续排序，没有则是已经有序，结束排序。

复杂度分析：

  • 时间复杂度：
    最好情况：正序情况下，比较(n-1)次，交换0次，时间复杂度O(n)
    最坏情况：反序情况下，比较n*(n-1)/2次，交换同样多的次数，时间复杂度为O(n^2)
    平均情况：时间复杂度O(n^2)

  • 空间复杂度：原地排序，辅助空间为O(1)，交换的时候用到了辅助空间

评价：优化后的交换排序思路简单，对小样本性能也较好，是好用的排序。

代码实现：设置一个标记位FLAG，每次外层循环设为FALSE，在内层循环中，若有一次交换就设置为TRUE。

代码：

/**
 * 改进的冒泡排序。可以避免冗余的比较和交换
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] modified_bubble_sort(int[] array) {
    int length = array.length;
    boolean flag = true;

    for (int i = 0; i < length - 1 && flag; i++) {
        flag = false;
        for (int j = 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
                flag = true;
            }
        }
    }
    return array;
}

2.2.3 鸡尾酒排序（Cocktail Sort）

思想：鸡尾酒排序，也叫双向冒泡排序(Bidirectional Bubble Sort)，这一下就很清晰了。以升序排列为例，
它先把最大的数移到序列末尾，然后接着从还没排好序的序列从尾部开始，把最小的数字移到最前边开头的地方。来回这样序列长度的一半的次数，排序就完成了

算法复杂度
时间复杂度：最好O(n)，最差O(n^2)，平均O(n^2)

空间复杂度：最差O(1)

代码：

/**
 * 双向冒泡排序
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] cocktailSort(int[] array) {
    int length = array.length;
    for (int i = 0; i < (length - 1) / 2; i++) {
        // 先把最大的数移到后边
        for (int j = i + 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
            }
        }

        // 再把最小的数移到最前边
        for (int k = length - i - 1; k > i; k--) {
            if (array[k] < array[k - 1]) {
                swap(array, k, k - 1);
            }
        }
    }
    return array;
}

/**
 * 改进的双向冒泡排序。可以避免冗余的比较和交换
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] optimizedCocktailSort(int[] array) {
    int length = array.length;
    boolean flag = true;

    for (int i = 0; i < (length - 1) / 2 && flag; i++) {
        flag = false;
        for (int j = i + 1; j < length - i;j++) {
            if (array[j] < array[j - 1]) {
                swap(array, j - 1, j);
                flag = true;
            }
        }

        // 若没有交换就直接跳出循环
        if (!flag) {
            break;
        }

        flag = false;
        for (int k = length - i - 1; k > i; k--) {
            if (array[k] < array[k - 1]) {
                swap(array, k, k - 1);
                flag = true;
            }
        }
    }

    return array;
}

关键代码：

}

注释：对于头部冒泡和尾部冒泡两次循环，对于j,k值得设置十分重要，因为两者会相互干涉，所以在设置的时候就需要考虑到对方的影响，都是处理待排序序列，头部冒泡的起始节点是待排序序列的头部开始，对于第（i+1）次来看，也就是从位置i开始；尾部冒泡是从尾部开始，也就是(length - 1 - i)开始

2.2.4 奇偶排序（Odd-Even Sort）

思想：选取奇数或者偶数为起始位置，以2位比较长度，两两进行比较。

代码实现：比较从第一位开始，下标能被2整除就是奇数排序，否则就是偶数排序，然后比较每个起始位置和下一个位置的大小，每次排序的增量为2，直到到达最后一位。

代码：

/**

 * 奇偶排序
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] oddEvenSort(int[] array) {
    int length = array.length;
    for (int i = 0; i < length; i++) {
        // 先判断是否能被2整除
        if (i % 2 != 0) {
            // 进行偶数排序
            for (int j = 2; j < length; j += 2) {
                if (array[j] < array[j - 1]) {
                    swap(array, j, j - 1);
                }
            }
        } else {
            // 进行奇数排序
            for (int j = 1; j < length; j += 2) {
                if (array[j] < array[j - 1]) {
                    swap(array, j, j - 1);
                }
            }
        }

    }
    return array;
}

/**

 * 改进的奇偶排序。可以避免冗余的比较和交换
 * @param array 待排序数组
 * @return 排好序数组
 */
public static int[] optimizedOddEvenSort(int[] array) {
    int length = array.length;
    int i;
    boolean sorted = true;

    while (sorted){
        sorted = true;
        for (int odd_even = 0; odd_even < 2; odd_even++) {
            for (i = odd_even; i < length - 1; i += 2) {
                if (array[i + 1] < array[i]) {
                    swap(array, i + 1, i);
                    sorted = false;
                }
            }
        }
    }

    return array;
}

private static void swap(int[] array, int i, int j) {
    if (i != j) {
        int temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }
}

2.2.5 快速排序（Quick Sort） — 最重要的排序算法

定义：对冒泡排序的改进。

思想：通过一趟排序实现对记录的分割，其中一部分记录的关键字均比另一部分记录的关键字小。则可分别对两部分记录进行排序，已达到整体有序的。

2.3 选择排序

2.3.1 简单选择排序

思想：每一趟从未排序元素中选择出一个最小（最大）的值，放在队头。

复杂度分析：
时间复杂度：最好，最坏比较都是(N-1)*(N+1)/2,交换是N-1次，所以时间复杂度是O(n^2)
空间复杂度：原地排序，一个临时变量，所以是常数级别的O(1)
稳定性:因为是相邻元素的比较，而且中途没有交换，所以是稳定的，而且是特别稳定，任何情况的比较和交换（不算赋值过程）都是一样的

评价：交换的次数较少，但是比较的次数较多，普通性能的排序。

代码实现：使用两层循环，每次用min（max）保存一个最值，对未排序元素进行依次比较，得出的结果与未排序元素头元素进行交换，然后又指针（数组下标）往后移动一位，形成新的无序元素表，循环直到剩余元素为空。

代码：

/**
 * 简单选择排序，第i轮循环记录一个最小值的位置，在循环结束后将位置为i的元素与最小值交换
 * @param array 待排序序列
 * @return 排好序序列
 */
public static int[] simple_select_sort(int[] array) {
    int length = array.length;
    int pos;

    for (int i = 0; i < length - 1; i++) {
        pos = i;
        for (int j = i + 1; j <= length - 1;j++) {
            if (array[j] < array[pos]) {
                pos = j;
            }
        }
        // 把最小元素换到队头来
        swap(array, i, pos);
    }
    return array;
}

private static void swap(int[] array, int i, int j) {
    if (i != j) {
        int temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }
}

##2.3.2 堆排序
储备知识：

  • 堆排序：
    1.堆排序是采用树形式的数据结构队数据进行排序的，其中每个堆（每次排序只用得上）都是完全二叉树。
    2.堆排序分为大根堆和小根堆 。大根堆（小根堆）表示在完全二叉树中，所用的非叶子节点都大于等于
    （小于等于）他们的左右子节点（存在）。所以堆的顶点都是最大（小）值，每次取出顶点的值，放在堆的尾部，形成有序序列。

  • 大根堆小根堆：
    用数组表示大\小根堆，按照从堆顶开始从上往下，从左往右存放到数组中。

  • 调整成大根堆：


    调整成最小堆正好相反

思想：1.建立初始大根堆或小根堆的原因是为了保证节点层次上的绝对有序，上一层的父节点一定大于下一层的子节点。为后面的调整简化了思想。
2.大根堆或者小根堆的顶点都是最大（小）值，每次取出顶点的值，放在堆的尾部，然后再将头部元素放到合适的位置，将次大或小的元素置于堆顶，依次重复以上步骤，形成有序序列。

复杂度分析：
运行时间主要在排序阶段。先是建立一个初始堆，需要时间O(n)（N/2-1）时间。然后第i次建立堆得比较和交换次数都是log2(i)次，而一共排序(n-1)次，所以排序的时间复杂度为O(nlogn)。
空间复杂度：O(1)，原地排序，辅助空间就一个temp

稳定性：
因为是父节点跟子节点的比较，不会连续，所以堆排序是不稳定的。

评价：堆排序是性能较好的排序，只是空间占用会大一点，不好理解一点。

代码实现：
设数组为a[0…N-1]
根据完全二叉树的性质：非叶子节点i的左右叶子节点（存在）下标分别为2i+1,2i+2。数组的最后一个非叶子节点为N/2-1。
实现堆排序步骤如下：
1.得到大根堆或者小根堆。这里以大根堆为例：
我们可以把每个非叶子节点都看成一个完全二叉树，然后都生成一个大根堆，得到一个初始大根堆堆。
2.取出生成大根堆的顶点值，与堆中第N个元素，也就是a[N-1]交换
3.此时大根堆被破坏，再重新生成大根堆，容量N–
4.重复2,3，直到N=1，排序完成

代码：

/**
  * 堆排序
  * @param array 待排序数组
  * @return
  */
 public static int[] heap_sort(int[] array) {
     int N = array.length;
     if (N < 2) {
         return array;
     }
     for (int i = N/2 - 1; i >= 0; i--) {
         fitToMaxHeap(array, i, N - 1);
     }
     for (int i = N - 1; i > 0; i--) {
         swap(array, 0, i);
         fitToMaxHeap(array, 0, i - 1);
     }
     return array;
 }

 /**
  * 调整成为大根堆
  * @param array
  * @param top 堆顶元素
  * @param tail 堆尾部元素
  */
 private static void fitToMaxHeap(int[] array, int top, int tail) {
     int max = array[top];
     // 沿着关键字较大的孩子节点向下
     for (int i = 2 * top + 1; i <= tail; i = 2 * i + 1) {
         if (i < tail && array[i + 1] > array[i]) {
             ++i;
         }
         // 如果这是最大的，直接跳出，如果不是，交换元素
         if (max >= array[i]) {
             break;
         }
         // 顶点也要移动
         array[top] = array[i];
         top = i;
     }
     array[top] = max;
 }

 private static void swap(int[] array, int i, int j) {
     if (i != j) {
         int temp = array[i];
         array[i] = array[j];
         array[j] = temp;
     }
 }

2.5 归并排序

2.5.1 原始的归并排序

预备知识：

  1. 归并在数据结构中的定义是：将两个或者两个以上的有序表组合成一个新的有序表。
  2. 归并算法不唯一：多路归并算法有多种实现它的路数是浮动的K，我们在这里讲的是最简单的二路归并。
  3. 分治法：将一个复杂的问题不断地分成两个或者更多相同或相似的子问题来解决，直到最后子问题可以直接简单求解为止，子问题的解得合并即为原问题的解。

思想：二路归并算法是利用分治法的思想。它假设初始序列有N个记录，将这N个元素看做N个有序子序列，每个子序列的长度为1，然后进行合并。

复杂度：
时间复杂度：首先， 归并排序需要将temp[0]…temp[n]中相邻长度为h的所有记录两两排序归，并存放到target[0]…target[n]里面去。此过程，需要将temp所有记录都扫描一遍，因此耗费O(n)的时间，而由完全二叉树的深度为[log2(n)]+1可知，归并排序需要进行[log2(n)]的常数倍次，因此，总的时间复杂度为O(nlog2(n))，而且是最好、平均、最坏都是如此。

空间复杂度：存放扫描后数据的空间为O(n)，扫描过程中递归需要的栈空间为O(logn)，所以时间复杂度为O(n+logn)。排序空间耗费很大，这是归并排序的一大缺陷。

稳定性：因为归并排序是相邻数字两两交换，所以说是稳定的。

评价：归并排序在外排序的时候用的更多一些，跟堆排序一样的缺点。

代码实现：两个步骤，先是划分成N个子序列，然后是两两合并。

代码：

/**
 * 归并排序
 * @param array 待排序序列
 * @return 已排序序列
 */
public static int[] merge_sort(int[] array) {
    int length = array.length;
    if (length <= 1) {
        return array;
    }
    length = array.length;
    int[] target = new int[length];

    return m_sort(array, target, 0, length - 1, length);
}

/**
 * 归并排序算法
 * @param array 待排序序列
 * @param target 排好序输出的目标序列
 * @param start 排序起始位置
 * @param end 排序末尾位置
 * @param length 待排序序列长度
 * @return 排好序序列
 */
private static int[] m_sort(int[] array, int[] target, int start, int end, int length) {
    int middle;
    int[] temp = new int[length];

    if (start == end) {
        target[start] = array[start];
    } else {
        middle = (start + end) / 2;

        m_sort(array, temp, start, middle, length);
        m_sort(array, temp, middle + 1, end, length);
        // 两两归并
        merge(temp, target, start, middle, end);
    }

    return target;
}

/**
 * 将数组temp[start...middle]和temp[middle+1...end]归并起来
 * 其实就是讲两个序列有序地合并，先在两个序列都不为空的情况下进行元素比较，
 * 哪个小插入哪个到尾部，直到其中至少一个序列为空，然后将剩余的序列依次插入尾部
 * 这里面的两个序列就来自于数组temp的前半（start...middle）和后半（middle+1...end）
 *
 * @param temp 待排序总序列
 * @param target 排好序之后输出的序列
 * @param start 前半序列起始位置
 * @param middle 中间分隔两个序列的位置，前半序列的中点位置，后半序列起始位置的前一个位置
 * @param end 后半序列的末尾位置
 */
private static void merge(int[] temp, int[] target, int start, int middle, int end) {
    int i = start;
    int j = middle + 1;
    int k = start;// target指针

    // 有序地合并序列，特别注意，这里target的插入起始位置跟temp的起始位置是一样的
    while (i <= middle && j <= end) {
        if (temp[i] < temp[j]) {
            target[k++] = temp[i++];
        } else {
            target[k++] = temp[j++];
        }
    }

    // 若前半没合并完，依次插入尾部
    while (i <= middle) {
        target[k++] = temp[i++];
    }

    // 若后半没合并完，依次插入尾部
    while (j <= end) {
        target[k++] = temp[j++];
    }
}

2.5.2 改进的归并排序

问题的提出：因为空间耗费大是归并排序的一大缺陷，而且这一缺陷是递归造成的。所以在这里，可以将递归优化成迭代。减少空间占用。

思想：将递归换成迭代，利用新创建的target数组与源数组辗转排序最终完成排序。

代码：

/**
 * 归并排序算法。从最小的间距为2开始，对数组进行迭代排序
 * @param array 待排序序列
 * @return 排好序序列
 */
private static int[] modified_m_sort(int[] array) {
    int interval = 1;
    int length = array.length;
    int[] target = new int[length];

    // 之所以调用两次。1.非原地排序，需把两个序列来回倒换
    // 2.即使interval长度超过了序列总厂，也有判断可以避免错误indexOutOfRange
    while (interval < length) {
        mergePass(array, target, interval, length);
        interval *= 2;

        if (interval > length) {
            return target;
        } else {
            mergePass(target, array, interval, length);
            interval *= 2;
        }

    }

    return array;
}

/**
 * 通过对序列里面的子序列进行排序进行，从头到尾，将标准长度的子序列两两归并
 * @param srcArray 待排序序列
 * @param targetArray 排好序序列
 * @param interval 每个归并子序列标准长度
 * @param length 待排序序列总长度
 */
private static void mergePass(int[] srcArray, int[] targetArray, int interval, int length) {
    int i = 0;
    int j;
    // 以两个间隔为单位，进行归并排序
    while (i <= length - 1 - (2 * interval - 1)) {
        merge(srcArray, targetArray, i, i + interval - 1, i + 2 * interval - 1);

        i += 2 * interval;
    }

    // 还剩不止一个子序列，进行两两归并
    if (length - 1 - i > interval - 1) {
        merge(srcArray, targetArray, i, i + interval - 1, length - 1);
    } else {
        // 最后若只剩下单个子序列，直接插入尾
        for (j = i; j < length; j++) {
            targetArray[j] = srcArray[j];
        }
    }
}

/**
 * 将数组temp[start...middle]和temp[middle+1...end]归并起来
 * 其实就是讲两个序列有序地合并，先在两个序列都不为空的情况下进行元素比较，
 * 哪个小插入哪个到尾部，直到其中至少一个序列为空，然后将剩余的序列依次插入尾部
 * 这里面的两个序列就来自于数组temp的前半（start...middle）和后半（middle+1...end）
 *
 * @param temp 待排序总序列
 * @param target 排好序之后输出的序列
 * @param start 前半序列起始位置
 * @param middle 中间分隔两个序列的位置，前半序列的中点位置，后半序列起始位置的前一个位置
 * @param end 后半序列的末尾位置
 */
private static void merge(int[] temp, int[] target, int start, int middle, int end) {
    int i = start;
    int j = middle + 1;
    int k = start;// target指针

    // 有序地合并序列，特别注意，这里target的插入起始位置跟temp的起始位置是一样的
    while (i <= middle && j <= end) {
        if (temp[i] < temp[j]) {
            target[k++] = temp[i++];
        } else {
            target[k++] = temp[j++];
        }
    }

    // 若前半没合并完，依次插入尾部
    while (i <= middle) {
        target[k++] = temp[i++];
    }

    // 若后半没合并完，依次插入尾部
    while (j <= end) {
        target[k++] = temp[j++];
    }
}

复杂度：
因为没有使用递归，所以这里的空间复杂度降低了，但是在interval最大的一次归并还是需要扫描整个序列的一半以上，所以空间复杂度仍然是O(n)，但是相比于递归已经好很多了。
时间复杂度：因为i是以2为底的指数，所以时间复杂度和改进之前的一样，都是O(nlogn)。所以，我们一半使用的归并排序都是非递归形式的归并排序。

3 外排序

  • 定义：外排序（External sorting）是指能够处理极大量数据的排序算法。

通常来说，外排序处理的数据不能一次装入内存，只能放在读写较慢的外存储器（通常是硬盘）上。外排序通常采用的是一种“排序-归并”的策略。在排序阶段，先读入能放在内存中的数据量，将其排序输出到一个临时文件，依此进行，将待排序数据组织为多个有序的临时文件。尔后在归并阶段将这些临时文件组合为一个大的有序文件，也即排序结果。

附加1：内省排序

附加2：计数排序&基数排序&桶排序

储备知识：
1.基于比较的排序算法是不能突破O(NlogN)的。简单证明如下：
N个数有N!个可能的排列情况，也就是说基于比较的排序算法的判定树有N!个叶子结点，比较次数至少为log(N!)=O(NlogN)(斯特林公式)。

非基于比较的排序：
计数排序（counting sort)
桶排序 （bucket sort)
基数排序（radix sort）

2.桶排序是一种思想，不是一种实现。故实现了桶排序思想的计数排序和基数排序都不属于比较排序。

计数排序

预定义：这里有一个预定义，数组里面的所有值都是自然数，也就是非负整数。
思想：找到待排序数组中所有数值的范围，根据范围创建一个计数数组，范围(min, max)。
然后求范围内各个数值的出现频率，然后在求其计数和，对数组中存在的数，此计数和即为在有序数组 中的末尾位置+1的值，最后反序根据数组输出有序数组。

复杂度分析：

时间复杂度：计数时间复杂度为O(N)，然后整理时间复杂度为O(N)，求计数和的时间复杂度为O(K)，所以总时间复杂度为O(N + k)，看K的大小，如果较小，则趋近O(N)，如果趋近于无穷大，时间复杂度会非常大，此法不再好用。

空间复杂度：计数数组为O(K)，结果数组O(N)，总空间复杂度为O(N + k)

从1开始计数版本，代码：

private static void countingSort(int[] A, int[] B, int k) {
    int[] C = new int[k];
    // 计数
    for (int j = 0; j < A.length; j++) {
        int a = A[j];
        C[a] += 1;
    }

    // 求计数和，此计数和就是数字在有序数组中的（位置+1）
    for (int i = 1; i < k; i++) {
        C[i] = C[i] + C[i - 1];
    }

    // 整理
    for (int j = A.length - 1; j >= 0; j--) {
        int a = A[j];
        B[C[a] - 1] = a;
        C[a] -= 1;
    }
}

优化计数数组版本代码：

 /**
 * 计数排序。
 * 先得出待排序数组中最大的数字，根据数字将放进计数数组。
 * 然后将计数数组依次累加得到数字当前的位置
 * 最后从待排序数组尾部开始读数字，放入结果数组。
 * 针对数组大小，优化过的计数排序
 * @param A 待排序数组
 * @param n 待排序数组长度
 * @return 排好序数组
 */
public static int[] countingSort(int[] A, int n) {
    if (n < 2) {
        return A;
    }

    // 0.计数数组的长度
    int max = A[0];
    int min = A[0];
    int length;

    for (int i = 1; i < A.length; i++) {
        if (max < A[i]) {
            max = A[i];
        }
    }

    for (int i = 1; i < A.length; i++) {
        if (min > A[i]) {
            min = A[i];
        }
    }

    length = max - min + 1;
    int[] countingArray = new int[length];
    int[] orderedArray = new int[n];

    int number;
    // 1.计数。先把每个数字的频率记录下来，放入计数数组
    for (int i = 0; i < n; i++) {
        number = A[i];
        countingArray[number - min]++;
    }

    // 2.求计数和。再把每个数字的频率累加，放入计数数组，就是当前数字的位置
    for (int j = 1; j < length; j++) {
        countingArray[j] = countingArray[j] + countingArray[j - 1];
    }

    // 3.从最后一位开始，放入结果数组，随着待排序数组数字放入结果数组，计数数组的放入位置应当减1
    for (int k = n - 1; k >= 0; k--) {
        number = A[k];
        // 数组实际位置和个数直接相差1，所以要减1
        orderedArray[--countingArray[number - min]] = number;
    }

    return orderedArray;
}

基数排序

预定义：集合里面是自然数。
思想：仍然是桶排序的思想，但是分进制来，例如基数为十进制的排序，那么就是：
1.选取一个长度为10的计数数组，每一个数字分别对应0-9
2.然后从高位或者低位开始，分别对每一位进行排序进行排序

代码实现：
1.找到数组最大值的位数
2.根据位数决定循环排序次数
3.接着每一次排序循环都是一个计数数组为0-9的计数排序

复杂度分析：
时间复杂度：被进制决定，因为这里是10进制，所以是ln，但是别的B进制就logB ，那么这里的时间复杂度是O(lnk * N)
空间复杂度：同计数排序，O(N + k)

代码：

/**
 * 基数排序，就是计数排序的多位数版本。桶排序的常见实现。
 * 有MSD和LSD两种实现，我们这里写的是最为通用的LSD。
 * 1.先判断数组最大数字的位数
 * 2.根据位数，进行从低位到高位的循环
 * @param A 待排序数组
 * @param n 待排序数组长度
 * @return 排好序数组
 */
public static int[] radixSort(int[] A, int n) {
    if (n < 2) {
        return A;
    }

    // 1.先找出最大位数
    int maxBit = findMaxBit(A, n);
    int radix = 1;
    int i;
    int j;

    // 临时数组和计数数组，计数数组是十进制的
    int[] temp = new int[n];
    int[] count = new int[10];

    // 2.根据位数，进行从低位到高位的循环
    for (i = 1; i <= maxBit; i++) {
        // 将技术数组重新初始化为0
        for (j = 0; j < 10; j++) {
            count[j] = 0;
        }
        // 将待排序数的尾缀数字0-9出现频率放进计数数组
        for (j = 0; j < n; j++) {
            int k = (A[j] / radix) % 10;
            count[k]++;
        }

        // 求计数和，也就是所有数字的理论位置
        for (j = 1; j < 10; j++) {
            count[j] = count[j - 1] + count[j];
        }

        // 根据数字的实际位置来存放到临时数组
        for (j = n - 1; j >= 0; j--) {
            int k = (A[j] / radix) % 10;
            temp[--count[k]] = A[j];
        }

        // 将排好序的数字返回给原数组
        for (j = 0; j < n; j++) {
            A[j] = temp[j];
        }

        // 进位
        radix *= 10;
    }

    return A;
}

private static int findMaxBit(int[] A, int n) {
    int max = A[0];
    int d = 1;
    // 先找最大值
    for (int i = 0; i < n; i++) {
        if (A[i] > max) {
            max = A[i];
        }
    }

    while (max >= 10) {
        max /= 10;
        d++;
    }

    return d;
}
