快速排序算法

1 传统的快速排序

1.1 基本概念

前言：因为快速排序是排序算法中最为重要，泛用性极高的算法。而且变体和改进还特别多，在此我就单独成章，把它拿出来说一说。

定义：快速排序是对冒泡排序的改进。因为他们都是通过扩大（升序排序中）数组的一半确定小于（冒泡排序是大于）另一半，来完成排序的。

思想：通过一趟排序实现将记录分割成独立的两部分，其中一部分记录的关键字均比另一部分记录的关键字小。则可分别对两部分记录进行排序，已达到整体有序的。

复杂度分析：
最优情况：
时间复杂度：最优情况下，每次都很均匀地分成两组，第一次划分比较的次数，每一组是n/2，那么两组就是n次，而递归地次数是O([long2n]+ 1)，设时间复杂度函数为T(x)，可得推导式：
T（n）≤2T（n/2） +n，T（1）=0
T（n）≤2（2T（n/4）+n/2） +n=4T（n/4）+2n
T（n）≤4（2T（n/8）+n/4） +2n=8T（n/8）+3n
……
T（n）≤n（2T（1） + n）+（log2n - 1）×n= O(nlogn)
所以时间复杂度为O(nlog2n).

空间复杂度：
每次递归都需要1个额外空间，所以空间复杂度为O(log2n)。

最坏情况：
时间复杂度：最坏情况下，序列是反序的，每次分组都是比上一次少划分一个记录的子序列，另一边则为空，这样就需要进行n-1次分组，递归树深度为n-1，每次比较了n-i次，一共比较了n*(n-1)/2次，所以时间复杂度为O(n^2)

稳定性：因为不是相邻交换了，所以快速排序是不稳定的。

评价：快速排序，大家说好，就是真的好。

1.2 代码实现

代码实现：

  • 找到枢纽位置pivot是关键步骤，取低位元素作为枢纽值，进行循环分组。
  • 每一趟分组，先从高位进行分组，选出比枢纽值小的值，与枢纽原位置进行交换（枢纽值处于高位）；接着低位开始进行分组，选出比枢纽值大的元素，与枢纽值进行交换（枢纽值处于低位），一趟结束。
  • 若低位仍然小于高位，就继续循环，直到低位与高位相等，循环结束

代码：

/**
 * 快速排序。分治法的应用，先将数组根据第一个元素大小分成两边。
 * 一边小，一边大。再对将两边的数组分别进行同样的划分。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 排好序数组
 */
public static int[] quick_sort(int[] array, int low, int high) {
    int pivot;

    if (low < high) {
        pivot = getPartition(array, low, high);
        quick_sort(array, low, pivot - 1);
        quick_sort(array, pivot + 1, high);
    }

    return array;
}

/**
 * 分组。根据第一个元素大小分成两边的数组，一边小，一边大。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 划分好的数组
 */
private static int getPartition(int[] array, int low, int high) {
    int pivot_key = array[low];

    // 从表的两端交替向中间扫描
    while (low < high) {
        while (low < high && array[high] >= pivot_key) {
            high--;
        }

        swap(array, low, high);

        while (low < high && array[low] <= pivot_key) {
            low++;
        }

        swap(array, low, high);
    }

    return low;
}

private static void swap(int[] array, int i, int j) {
    if (i != j) {
        int temp = array[i];
        array[i] = array[j];
        array[j] = temp;
    }
}

2. 改进的快速排序

2.1 快速排序的问题

问题:最直观的快排的问题出在它的递归实现上面，一旦数据量上去之后，而枢纽值选取不合理的话，很有可能出现线性次数的递归，然后产生爆栈（栈溢出异常）。次要问题是，在数据量太小的时候又性能一般，不如插入排序等等这些基础排序算法。

以上这些内容细分可以分为四个问题，我在这里一个一个解决：

1) 枢轴值选取不合理。
 问：为什么在正序或者反序的时候，快速排序的时间复杂度为平方阶O(n^2)？
 答： 因为对于这两种情况，一边的数组是空的，另一边又是待排序数组的完整版。而且分完组的话枢纽值要么最终在头上，要么最终在尾巴上，比较次数和比较的循环轮数都是最大的。最后的时间复杂度达到了O(n^2)。

2) 递归交换的复杂度较高
 问：这里面的分组函数getPartition每次调用swap都会进行3次操作，能不能改成替换，只进行一次操作？
 答：可以。

3) 性能问题（小数据样本时）
 问：当数组数量低于7个（资料显示）的时候，采用插入排序反而效率较高？
 答：博士生去小学教“1+1=2”也不见得比本科生教得好。快速排序对于基本有序，或者样本极少的数组排序的确效率不高。插入排序反而比较适合基本有序的小样本排序。

4) 递归占用太多栈空间
 问：大家知道，递归对性能是有一定影响的，quick_sort函数在其尾部有两次递归操作。如果待排序的序列划分极端不平衡（正序或反序），递归深度将趋近于n，而不是平衡时的log2n，栈的空间是极其有限的，如何减少递归？
 答：解决递归地最好方法就是使用迭代。所以我们需要优化这里面不必要的递归交换。甚至直接采用非递归的方法实现快速排序。

2.2 快速排序的改进方案

1) 枢轴值不合理解决方案
修改getPartition函数。枢轴不合理，那就选取合理的枢轴。现有的方案中：随机选取比较耗费性能，而且还不能保证一定会简化排序；九数取中分三路进行三路取中，较为复杂，先不讨论；这里采用三数取中选取枢轴值。

/**
 * 分组。先进行三数取中选取枢轴值，再根据枢轴值分成两个数组，一边小，一边大。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 划分好的数组
 */
private static int getPartition1(int[] array, int low, int high) {
    int pivot_key;
    // 进行三数取中
    int mid = low + (high - low) / 2;
    // 先确保左边是比右边小
    if (array[low] > array[high]) {
        swap(array, low, high);
    }
    // 再确保中间比右边小
    if (array[mid] > array[high]) {
        swap(array, mid, high);
    }
    // 最后确保左边比中间小
    if (array[low] > array[mid]) {
        swap(array, low, mid);
    }
    pivot_key = array[low];

    // 从表的两端交替向中间扫描
    while (low < high) {
        while (low < high && array[high] >= pivot_key) {
            high--;
        }

        swap(array, low, high);

        while (low < high && array[low] <= pivot_key) {
            low++;
        }

        swap(array, low, high);
    }

    return low;
}

}

2.2)递归交换的复杂度较高解决方案1（左老师方法—哨兵法）
或者换一种思路，设计一个小于等于枢轴值的区域，先将取出的枢轴与数组末尾进行交换，设为哨兵。然后将所有小于等于枢轴值的数字都放进区域，并将区域末尾指针相应右移，直到整个数组末尾。

     // 每次通过划分函数确定一个位置，然后让剩余的数组分为两边，又继续确定位置，直到所有位置都确定下来
    public static int[] quick_sort(int[] array, int low, int high) {
    int pivot;

    // 不同于普通快排的是，此方法会将要确定位置的中枢值先置于数组尾部，成为哨兵
    // 排序对比的时候从头部开始，每个值大小和哨兵比较，而在比较数组尾部之前，都不用移动哨兵
    if (low < high) {
        pivot = getPartition(array, low, high);

        quick_sort(array, low, pivot - 1);
        quick_sort(array, pivot + 1, high);
    }

    return array;
}

/**
 * 划分方法。完全按照左老师的方法来。
 * 1.交换枢划分值和数组高位（尾部）值，让划分值成为哨兵
 * 2.先在规定区域(high, low)的低位前一位规定一个小于等于划分值区域，初始长度为0
 * 3.移动index指针，遇到满足条件的数，就放在小于等于划分值区域的下一位（就是交换），区域长度          增加1，直到指针走到高位
 *
 * 虽然较好的去划分值的方法是三数取中，但是随机数也是可以的。
 * @param A 待排序数组
 * @param low 低位指针
 * @param high 高位指针
 * @return 划分值在数组中位置以及一个划分过的数组
 */
private static int getPartition(int[] array, int low, int high) {
    // 1.交换枢划分值和数组高位（尾部）值，让划分值成为哨兵
    swap(array, low, high);
    // 2.先在规定区域(high, low)的低位前一位规定一个小于等于划分值区域，初始长度为0
    int smallerTail = low - 1;
    int current = low;
    // 3.移动index指针，遇到满足条件的数，就放在小于等于划分值区域的下一位（就是交换），区域         // 长度增加1，直到指针走到高位
    while (current <= high) {
        if (array[current] <= array[high]) {
            swap(array, ++smallerTail, current);
        }
        current++;
    }
    return smallerTail;
}

4.1) 爆栈或者内存溢出问题解决方案1
修改quick_sort函数。将if改成while，使用迭代，删除第二次递归，然后将之前荒废掉的low利用起来，作为指针使用。

/**
 * 改进的快速排序。优化了递归。
 * @param array 待排序数组
 * @param low 低位下标
 * @param high 高位下标
 * @return 排好序数组
 */
public static int[] modified_quick_sort2(int[] array, int low, int high) {
    int pivot;

    // 将递归换成迭代，减少对栈的使用
    while (low < high) {
        pivot = getPartition(array, low, high);
        quick_sort(array, low, pivot - 1);
        low = pivot + 1;
    }

    return array;
}

4.2) 爆栈或者内存溢出问题解决方案2：
使用非递归方法即可解决。这里的非递归就是把系统函数栈换成我们自己申请的额外空间栈。实现一样的算法，具体代码如下：

/**
 * 非递归实现，使用一个栈，用每次划分区域得出的中枢值将区域进行再次划分，
 * 直到中枢值等于区域的头或者尾部为止
 * @param array 待排序数组
 * @param head 数组头部
 * @param tail 数组尾部
 * @return 一个排好序的升序数组
 */
public static int[] quick_sort2(int[] array, int head, int tail) {
    int pivot;
    // 用一个栈来实现，每次划分只排一个数的序，就是pivot中枢值，当局部头尾等于pivot，
    // 就将其出栈，当得出的中枢 值和局部的头尾值不相等时，就将不等部分入栈，
    // 直到全部数字都出栈为止
    Stack<Integer> stack = new Stack<>();
    pivot = getPartition(array, head, tail);

    if (head < pivot - 1) {
        stack.push(head);
        stack.push(pivot);
    }
    if (tail > pivot + 1) {
        stack.push(pivot + 1);
        stack.push(tail);
    }

    while (!stack.isEmpty()) {
        int high = stack.pop();
        int low = stack.pop();

        pivot = getPartition(array, low, high);
        if (low < pivot - 1) {
            stack.push(low);
            stack.push(pivot - 1);
        }
        if (high > pivot + 1) {
            stack.push(pivot + 1);
            stack.push(high);
        }
    }

    return array;
}

// 分隔算法也做了小小的改动
private static int getPartition(int[] array, int low, int high) {
    int smallerTail = low - 1;
    int current = low;
    // 就是这一句，将分隔值与数组高位进行交换
    swap(array, low, high);
    // 如果找到小于中枢值的数，放到较小区域
    while (current <= high) {
        if (array[current] <= array[high]) {
            swap(array, ++smallerTail, current);
        }
        current++;
    }
    return smallerTail;
}

1) 最终优化版快速排序（非递归）

/**
 * 非递归实现，使用一个栈，用每次划分区域得出的中枢值将区域进行再次划分，
 * 直到中枢值等于区域的头或者尾部为止
 * @param array 待排序数组
 * @param head 数组头部
 * @param tail 数组尾部
 * @return 一个排好序的升序数组
 */
public static int[] quick_sort2(int[] array, int head, int tail) {
    int pivot;

    // 用一个栈来实现，每次划分只排一个数的序，就是pivot中枢值，当局部头尾等于pivot，
    // 就将其出栈，当得出的中枢 值和局部的头尾值不相等时，就将不等部分入栈，
    // 直到全部数字都出栈为止
    Stack<Integer> stack = new Stack<>();
    pivot = getPartition(array, head, tail);

    if (head < pivot - 1) {
        stack.push(head);
        stack.push(pivot);
    }
    if (tail > pivot + 1) {
        stack.push(pivot + 1);
        stack.push(tail);
    }

    while (!stack.isEmpty()) {
        int high = stack.pop();
        int low = stack.pop();

        pivot = getPartition(array, low, high);
        if (low < pivot - 1) {
            stack.push(low);
            stack.push(pivot - 1);
        }
        if (high > pivot + 1) {
            stack.push(pivot + 1);
            stack.push(high);
        }
    }

    return array;
}

private static int getPartition(int[] array, int low, int high) {
    //1.先三数取中选好枢轴值
    int pivot_key;
    // 进行三数取中
    int mid = low + (high - low) / 2;
    // 先确保左边是比右边小
    if (array[low] > array[high]) {
        swap(array, low, high);
    }
    // 再确保中间比右边小
    if (array[mid] > array[high]) {
        swap(array, mid, high);
    }
    // 最后确保左边比中间小
    if (array[low] > array[mid]) {
        swap(array, low, mid);
    }
    pivot_key = array[low];
    swap(array, pivot_key, high);

    // 2.划分出一块默认长度为0，指针指向数组区域前前一项的空间作为存放小于枢轴值的区域
    int smallerTail = low - 1;
    int current = low;
    // 3.一直比较，直到最后一个枢轴值被比较，完成分割
    while (current <= high) {
        if (array[current] <= array[high]) {
            swap(array, ++smallerTail, current);
        }
        current++;
    }
    return smallerTail;
}

        array[low] = array[high];

        while (low < high && array[low] <= pivot_key) {
            low++;
        }
        array[high] = array[low];
    }
    array[low] = temp;

注释：这里用到了插入排序的思想

2.3.7 快速排序的单链表实现（C语言版）

背景：因为看到有的阿里面试面经里面写到了，觉得很有趣，所以就在这里收录下来了。

思路：C里面都是操作指针，快排也有操作指针实现的方法。但是这里用改变值得方法来排序，会不会更接近Java程序员的思维？对于初始版本，选择，采用两个指针，一快一慢。快指针负责比较，慢指针负责记录比枢轴小的序列的末尾位置，这个位置后来会存放枢轴。一直循环直到快指针指向数组末尾指针的next—NULL。

具体实例：

/**
 * 最优解：将链表分成两个个部分，小于等于，大于，然后依次链接上去
 * @param head
 * @param val
 * @return
 */
public ListNode getPartition(ListNode head, int val) {
    if (head == null || head.next == null) {
        return head;
    }

    ListNode smallerHead = new ListNode(-1);
    ListNode smallerTail = smallerHead;
    // 这里指向head是为了避免小于值链表为空的时候，链表smallerHead.next为空，需要对返回结果进行分类讨论
    smallerTail.next = head;

    ListNode largerHead = new ListNode(-1);
    ListNode largerTail = largerHead;

    ListNode cur = head;
    while (cur != null) {
        if (cur.val > val) {
            largerTail.next = cur;
            largerTail = largerTail.next;
            // 处理小于值的链表，让小于指针next始终保持在链表current一致
            smallerTail.next = cur.next;
        } else {
            // 这里有奇效，只需要移动过去就可以了，且最后一位会自动置空
            smallerTail = cur;
        }
        cur = cur.next;
    }

    // 大于值的链表需要置空
    largerTail.next = null;
    smallerTail.next = largerHead.next;
    return smallerHead.next;
}

实际应用：TOP-K问题
对于TOP-K问题，之前只是觉得这是个简单的快排取编号的问题，上网一搜才发现这个问题不仅仅有至少7种解法，而且还有可以用来处理海量数据。
题目1：设计一组N个数，确定其中第k个最大值。
解法：

// 递归解法，调用了已有的快排的分隔函数getPartition，十分简洁

public static int findkMax(int[] arr, int l, int r, int k) {
    int pivot = QuickSortMod.getPartition(arr, l, r);
    if (pivot < k) {
        return findkMax(arr, pivot + 1, r, k);
    } else if (pivot > k){
        return findkMax(arr, l, pivot - 1, k);
    }

    return k;

}

// 非递归解法，直接得出最终结果值，但是步骤较多

   public static int findKMax2(int[] arr, int l, int r, int k) {

    int pivot = QuickSortMod.getPartition(arr, l, r);
    Stack<Integer> stack = new Stack<>();

    if (k < pivot) {
        stack.push(l);
        stack.push(pivot - 1);
    }
    if (k > pivot) {
        stack.push(pivot + 1);
        stack.push(r);
    }

    while (pivot != k || !stack.isEmpty()) {
        int high = stack.pop();
        int low = stack.pop();

        pivot = QuickSortMod.getPartition(arr, low, high);
        if (k < pivot) {
            stack.push(low);
            stack.push(pivot - 1);
        } else if (k > pivot) {
            stack.push(pivot + 1);
            stack.push(high);
        }
    }

    return arr[k];
}

题目2：海量数据处理的 Top K算法(问题) 小顶堆实现
问题描述：有N(N>>10000)个整数,求出其中的前K个最大的数。（称作Top k或者Top 10）

　　问题分析：由于(1)输入的大量数据；(2)只要前K个，对整个输入数据的保存和排序是相当的不可取的。

　　　　　　　 可以利用数据结构的最小堆来处理该问题。

　　　　　　　 最小堆如图所示，对于每个非叶子节点的数值，一定不大于孩子节点的数值。这样可用含有K个节点的最小堆来保存K个目前的最大值(当然根节点是其中的最小数值)。

　　　　　　每次有数据输入的时候可以先与根节点比较。若不大于根节点，则舍弃；否则用新数值替换根节点数值。并进行最小堆的调整。
